# 写作方法论库

16种写作方法的完整说明，含所有步骤和AI自动化适配方案。

---

## 1. 雪花法 (Snowflake Method)

### 原理
从最小的核心（一句话）出发，逐步扩展细节，像雪花从简单结构生长为复杂晶体。

### 10步流程

| 步骤 | 内容 | 字数 | AI适配 |
|------|------|------|--------|
| 1 | 一句话摘要 | 15字以内 | GPT生成5个→人工选1个 |
| 2 | 一段话摘要 | 5句：设置+3灾难+结局 | AI扩展步骤1 |
| 3 | 角色摘要 | 每角色1页：名字/目标/动机/冲突/顿悟/简述 | AI生成角色卡模板 |
| 4 | 扩展摘要 | 每句→一段(1页总) | AI将5句各扩展为段落 |
| 5 | 角色简述 | 每角色1页完整故事 | AI为每个角色写完整弧线 |
| 6 | 扩展大纲 | 4-5页完整摘要 | AI将段落扩展为完整大纲 |
| 7 | 角色图表 | 详细角色设定表 | AI生成结构化角色sheet |
| 8 | 场景表 | 每场景一行(POV/发生什么) | AI从大纲生成场景列表 |
| 9 | 场景描述 | 每场景几段描述 | AI扩展每个场景 |
| 10 | 初稿 | 完整初稿 | AI逐场景生成正文 |

### AI自动化流程
```
用户输入: 一句话核心概念
→ AI生成步骤1-4 (结构化扩展)
→ 人工审核/调整
→ AI生成步骤5-8 (角色和场景展开)
→ 人工审核/调整
→ AI生成步骤9-10 (正文)
→ 人工审核/修改
```

## 2. Save the Cat 节拍表 (Blake Snyder Beat Sheet)

### 原理
电影编剧方法论，15个固定节拍+精确百分比位置，保证叙事节奏。

### 15个节拍

| # | 节拍 | 百分比位置 | 内容 | 网文章节映射(100章) |
|---|------|-----------|------|---------------------|
| 1 | Opening Image | 0-1% | 开场画面，展示主角日常 | 第1章开头 |
| 2 | Theme Stated | 5% | 有人说出主题(主角不理解) | 第3-5章 |
| 3 | Set-Up | 1-10% | 建立世界和角色 | 第1-10章 |
| 4 | Catalyst | 10% | 触发事件，打破平衡 | 第10章 |
| 5 | Debate | 10-20% | 主角犹豫是否接受召唤 | 第10-20章 |
| 6 | Break into Two | 20% | 进入第二幕，新世界 | 第20章 |
| 7 | B Story | 22% | 副线开始(通常是爱情线) | 第22章 |
| 8 | Fun and Games | 20-50% | "承诺的前提"，最有趣的部分 | 第20-50章 |
| 9 | Midpoint | 50% | 中点，假胜利或假失败 | 第50章 |
| 10 | Bad Guys Close In | 50-75% | 反派力量加强 | 第50-75章 |
| 11 | All Is Lost | 75% | 最低点，最大失败 | 第75章 |
| 12 | Dark Night of the Soul | 75-80% | 主角反思，触底 | 第75-80章 |
| 13 | Break into Three | 80% | 找到解决方案 | 第80章 |
| 14 | Finale | 80-99% | 最终决战，应用学到的教训 | 第80-99章 |
| 15 | Final Image | 99-100% | 结尾画面，与Opening Image对比 | 第100章结尾 |

### AI适配
```
输入: 类型 + 核心设定
AI输出:
  - 15个节拍的具体内容
  - 各节拍对应的章节号
  - 每个节拍的1-2句描述
  - 情感强度曲线图
```

## 3. 三幕结构 (Three-Act Structure)

### 原理
故事分为设置/对抗/解决三个幕，比例25%/50%/25%。

### 结构详解

```
第一幕 - 设置 (25%)
├── 普通世界展示
├── 激励事件 (12%)
├── 第一转折点 (25%) → 进入第二幕
└── 建立：主角/目标/障碍/赌注

第二幕 - 对抗 (50%)
├── 上升行动
├── 中点 (50%) - 假胜利/假失败
├── 第一掐点 (37%) - 表面成功
├── 第二掐点 (62%) - 表面失败
├── 第二转折点/最低点 (75%)
└── 递增挑战和复杂化

第三幕 - 解决 (25%)
├── 高潮准备
├── 高潮 (90%)
├── 反转/启示
└── 新常态/余韵
```

### 6-8转折点

1. **激励事件** (12%): 打破平衡的事件
2. **第一转折点** (25%): 主角投入冒险
3. **第一掐点** (37%): 提高赌注
4. **中点** (50%): 方向改变
5. **第二掐点** (62%): 对手出击
6. **第二转折点/最低点** (75%): 一切似乎失败
7. **高潮** (90%): 最终对决
8. **结局** (100%): 新平衡

## 4. 英雄之旅 (Hero's Journey / Monomyth)

### 原理
约瑟夫·坎贝尔总结的神话原型结构，12阶段循环。

### 12阶段

| # | 阶段 | 英文 | 内容 |
|---|------|------|------|
| 1 | 普通世界 | Ordinary World | 展示主角的日常生活 |
| 2 | 冒险召唤 | Call to Adventure | 打破平衡的事件 |
| 3 | 拒绝召唤 | Refusal of the Call | 主角犹豫/恐惧 |
| 4 | 遇见导师 | Meeting the Mentor | 获得指导/工具/信心 |
| 5 | 跨越门槛 | Crossing the Threshold | 进入新世界 |
| 6 | 试炼/盟友/敌人 | Tests, Allies, Enemies | 新世界中的挑战 |
| 7 | 接近洞穴 | Approach to the Inmost Cave | 准备面对最大挑战 |
| 8 | 磨难 | The Ordeal | 最大的考验/死亡体验 |
| 9 | 奖赏 | Reward | 获得力量/知识/宝物 |
| 10 | 回归之路 | The Road Back | 开始回归 |
| 11 | 复活 | Resurrection | 最终考验/净化/转变 |
| 12 | 归来 | Return with the Elixir | 带着变化回到普通世界 |

### 8个原型

| 原型 | 功能 | 网文对应 |
|------|------|----------|
| 英雄 | 主角，经历成长 | 主角 |
| 导师 | 指导者 | 师父/长辈 |
| 门槛守卫 | 测试英雄的角色 | 试炼/守关 |
| 使者 | 带来召唤 | 触发事件人物 |
| 变形者 | 忠诚不确定 | 亦敌亦友 |
| 阴影 | 对手/反派 | 大反派 |
| 盟友 | 帮助者 | 队友/好友 |
| 骗子 | 喜剧角色/催化剂 | 搞笑角色/谋士 |

## 5. Story Grid

### 原理
Shawn Coyne开发，基于"五戒"和"类型公约"的编辑分析方法。

### 五戒 (Five Commandments)

每个故事/每个场景都必须有：
1. **激励事件** (Inciting Incident): 打破平衡
2. **转折复杂化** (Turning Point Progressive Complication): 情况恶化
3. **危机** (Crisis): 两难选择
4. **高潮** (Climax): 做出选择
5. **结局** (Resolution): 选择的后果

### 价值光谱

每个类型有自己的价值轴：
- 惊悚: 生→死/命运比死还惨
- 爱情: 爱→恨/孤独
- 动作: 生→死
- 犯罪: 正义→不正义
- 恐怖: 生→命运比死还惨(无力)
- 表演: 尊重→耻辱

### 必要场景 & 类型公约

每种类型都有"必须有"的场景和公约：
- 爱情：必须有"恋人分离"场景
- 惊悚：必须有"英雄被困"场景
- 犯罪：必须有"犯罪被揭露"场景

## 6. Dan Harmon 故事圈 (Story Circle)

### 原理
简化的英雄之旅，8步循环，强调对称性。

### 8步循环

```
        1. You (舒适区)
      /                \
  8. Change            2. Need (有需求)
    |                    |
  7. Return            3. Go (出发)
    |                    |
  6. Take              4. Search (寻找)
      \                /
        5. Find (找到)
```

| # | 步骤 | 关键词 | 对称关系 |
|---|------|--------|----------|
| 1 | You | 舒适区中的角色 | 与5对称(找到) |
| 2 | Need | 角色有需求/欲望 | 与6对称(付出代价) |
| 3 | Go | 进入不熟悉的情境 | 与7对称(回归) |
| 4 | Search | 适应并寻找 | 与8对称(已改变) |
| 5 | Find | 找到想要的 | 与1对称(起始) |
| 6 | Take | 付出代价得到 | 与2对称(满足需求) |
| 7 | Return | 回到熟悉的情境 | 与3对称(出发) |
| 8 | Change | 已经改变 | 与4对称(不再寻找) |

### 对称性原则
- 上半圈(1-4)是"下降"，下半圈(5-8)是"上升"
- 每一步与对面步骤形成镜像
- 可递归应用：整本书/每一卷/每一章/每个场景

## 7. 起承转合 (Ki-Shō-Ten-Ketsu)

### 原理
东亚叙事结构，不需要西方式冲突，强调展开和意外转折。

### 四部分

| 部分 | 日文 | 功能 | 比例 |
|------|------|------|------|
| 起 (Ki) | 起 | 介绍主题/人物/世界 | 25% |
| 承 (Shō) | 承 | 展开/发展/深入 | 25% |
| 转 (Ten) | 転 | 意外转折/新视角 | 25% |
| 合 (Ketsu) | 結 | 收束/余韵/启示 | 25% |

### 与西方结构的区别

- **不需要冲突**: "转"是视角转换，不是对抗
- **强调和谐**: 结局追求和谐而非胜负
- **留白**: "合"留有余韵，不全部说透
- **适合**: 日常系/治愈系/文艺类/散文体小说

## 8. Freytag 金字塔 (Five-Act Structure)

### 原理
Gustav Freytag分析戏剧的五幕结构，金字塔形状。

### 五幕

```
          高潮 (Climax)
         /      \
    上升动作    下降动作
   (Rising)    (Falling)
   /                \
阐述              结局
(Exposition)    (Denouement)
```

| 幕 | 名称 | 内容 | 位置 |
|----|------|------|------|
| 1 | 阐述 (Exposition) | 背景/人物/设定 | 0-20% |
| 2 | 上升动作 (Rising Action) | 冲突升级 | 20-50% |
| 3 | 高潮 (Climax) | 最高张力点 | 50% |
| 4 | 下降动作 (Falling Action) | 后果展开 | 50-80% |
| 5 | 结局 (Denouement) | 解决/新平衡 | 80-100% |

## 9. 七点结构 (Seven-Point Story Structure)

### 原理
Dan Wells提出，7个关键情节点，非线性设计顺序。

### 7个情节点

| # | 点 | 功能 | 位置 |
|---|---|------|------|
| 1 | Hook (开钩) | 起始状态，与结局对比 | 开头 |
| 2 | Plot Turn 1 (转折1) | 引入冲突/召唤 | 15-20% |
| 3 | Pinch Point 1 (掐点1) | 展示反派力量 | 30-35% |
| 4 | Midpoint (中点) | 从被动到主动 | 50% |
| 5 | Pinch Point 2 (掐点2) | 反派全力出击 | 65-70% |
| 6 | Plot Turn 2 (转折2) | 获得最后一块拼图 | 80-85% |
| 7 | Resolution (结局) | 最终解决 | 结尾 |

### 非线性设计顺序

Wells建议按以下顺序设计（而非时间顺序）：
1. 先定Resolution(结局)
2. 再定Hook(开头) — 必须与结局形成对比
3. 定Midpoint — 从被动到主动的转变
4. 定Plot Turn 1 — 如何从开头到中点
5. 定Plot Turn 2 — 如何从中点到结局
6. 定Pinch Point 1和2 — 压力施加点

## 10. Scene & Sequel + MRU

### 原理
Dwight Swain的微观叙事单元理论，每个叙事由Scene和Sequel交替组成。

### Scene (场景)

```
Scene结构:
  Goal (目标): 角色想要什么？
  → Conflict (冲突): 什么阻碍？
  → Disaster (灾难): 目标如何失败？(通常变得更糟)

示例:
  Goal: 林轩想要通过宗门考核
  Conflict: 竞争者暗中使绊、考题超出预期
  Disaster: 被判定不合格，面临被逐出宗门
```

### Sequel (续篇)

```
Sequel结构:
  Reaction (反应): 情感反应
  → Dilemma (困境): 面临什么选择？
  → Decision (决定): 做出什么决定？(下一个Scene的Goal)

示例:
  Reaction: 震惊、不甘、愤怒
  Dilemma: 接受结果离开？还是冒险挑战更高层裁决？
  Decision: 决定挑战长老裁决（→ 新Scene的Goal）
```

### MRU (Motivation-Reaction Units) 段落级别

```
更微观的单位(Jack Bickham):
  Motivation (外部刺激): 发生了什么（客观描述）
  → Reaction (内部反应): 角色如何反应
    - 感觉 (involuntary): "心一沉"
    - 反射动作: "下意识后退一步"
    - 情绪: "恐惧涌上心头"
    - 理性思考: "不对，这里不应该有人"
    - 语言/行动: "谁在那里？"

段落写作规则:
  - Motivation段(外部)和Reaction段(内部)交替
  - Motivation在前，Reaction在后
  - Reaction内部顺序：感觉→反射→情绪→理性→行动
```

### AI适配
```
AI写作时注入:
  "请按Scene结构写本段: 目标是[X]，冲突是[Y]，结果以灾难结束"
  "请按MRU规则: 先写外部刺激，再按感觉→反射→情绪→理性→行动顺序写反应"
```

## 11. 章尾钩子技术 (Cliffhanger Techniques)

### 9种钩子类型

| # | 类型 | 描述 | 示例 |
|---|------|------|------|
| 1 | 悬念型 | 提出未解答的问题 | "那个神秘人，究竟是谁？" |
| 2 | 危机型 | 主角陷入危险 | "毒雾逼近，解药就在对面——但桥断了" |
| 3 | 反转型 | 颠覆认知 | "她摘下面具，竟然是已经'死去'三年的师姐" |
| 4 | 承诺型 | 暗示精彩即将到来 | "明天，就是宗门大比的日子" |
| 5 | 情感型 | 强烈情感未得到回应 | "我喜欢你。" 她说完转身就走 |
| 6 | 信息型 | 透露关键新信息 | "这枚令牌上的纹路……和我父亲留下的一模一样" |
| 7 | 选择型 | 面临重大选择 | "左边是师父，右边是她——我只能救一个" |
| 8 | 时间型 | 倒计时/截止日期 | "距离秘境关闭，还有三天" |
| 9 | 预兆型 | 暗示危险/变化 | "天边的乌云，带着一丝不祥的血红" |

### 结构规则

- 每章必须以某种钩子结尾
- 避免连续使用同一类型
- 不要在钩子后立即解决（至少跨1章）
- 主线钩子和支线钩子交替使用
- 网文中：偶数章用强钩子（付费章节入口）

## 12. 网文黄金三章

### 原理
前三章决定读者留存，是网文的生死线。

### 第一章要求
- **开头50字**: 必须有冲突/悬念/信息差
- **第一章必须完成**:
  - 展示金手指（或暗示）
  - 建立主角的核心特征
  - 展示主角的处境（通常是困境）
  - 至少一个小爽点
  - 章尾钩子
- **禁止**: 大段背景介绍、纯对话水字数、无冲突的日常

### 第二章要求
- 金手指正式启动
- 第一次使用金手指获得优势
- 展示升级/变强的希望
- 建立第一个小目标
- 出现第一个对手/压力

### 第三章要求
- 第一次打脸（用金手指碾压看不起主角的人）
- 获得第一个奖励/收获
- 建立中期目标
- 读者能明确感受到"爽"

### AI适配
```
生成三章时的检查清单:
  □ 第1章: 50字内有冲突？有金手指？有爽点？有钩子？
  □ 第2章: 金手指启动？有优势展示？有目标？有压力？
  □ 第3章: 有打脸？有奖励？目标清晰？读者感到爽？
```

## 13. 爽点设计

### 8种爽点类型

| # | 类型 | 描述 | 读者情感 |
|---|------|------|----------|
| 1 | 打脸 | 主角碾压轻视自己的人 | 痛快/解气 |
| 2 | 升级 | 突破境界/获得新能力 | 兴奋/期待 |
| 3 | 获宝 | 获得稀有物品/传承 | 满足/惊喜 |
| 4 | 装逼 | 低调展示实力震惊众人 | 虚荣满足 |
| 5 | 打宝 | 击败强敌获得奖励 | 成就感 |
| 6 | 情感 | 情感满足(告白/认亲/报仇) | 感动/释放 |
| 7 | 反转 | 众人以为主角输了但其实赢了 | 惊喜/反差 |
| 8 | 碾压 | 以绝对实力差距轻松获胜 | 畅快 |

### 节奏控制模式

```
小爽: 每2000-3000字一个
  ├── 对话中的机智回击
  ├── 小收获(灵石/丹药)
  └── 细节上的优势展示

中爽: 每章(3000-5000字)一个
  ├── 小打脸
  ├── 阶段性胜利
  └── 新能力获得

大爽: 每3-5章一个
  ├── 境界突破
  ├── 大打脸(公开场合)
  ├── 重大获宝
  └── 关键战斗胜利

超级爽: 每卷/弧线高潮
  ├── 越级挑战大胜
  ├── 复仇成功
  ├── 惊天逆转
  └── 多条线索汇聚爆发
```

## 14. 升级体系

### 模板设计

```
标准等级模板(7-12大等级):
  凡人 → 炼气 → 筑基 → 金丹 → 元婴 → 化神 → 大乘 → 渡劫 → 仙人

每大等级 × 3-9小阶段:
  炼气一层 → 炼气二层 → ... → 炼气九层 → 筑基

附加维度:
  - 体修/法修/剑修 路线分化
  - 品质差异(低品金丹/中品/上品/极品)
  - 辅助系统(炼丹/炼器/阵法/符箓 等级)
```

### 升级节奏

```
网文升级节奏参考:
  - 低等级快速升级(5-10章一个小等级) → 给读者即时满足
  - 中等级正常节奏(10-20章一个小等级)
  - 高等级缓慢(20-50章一个小等级) → 制造期待感
  - 关键突破作为大爽点 → 通常配合大战/危机
  
  总字数映射(200万字完本为例):
    前50万字: 前4个大等级(从凡人到金丹)
    50-120万字: 第5-6个大等级(元婴化神)
    120-180万字: 第7-8个大等级(大乘渡劫)
    180-200万字: 最终等级+收官
```

## 15. 网文副本结构

### 标准副本/剧情卷结构 (10-20章弧线)

```
Phase 1: 入场 (1-2章)
  - 进入新场景/新副本/新区域
  - 展示环境和规则
  - 明确本卷目标

Phase 2: 探索 (2-4章)
  - 收集信息
  - 遇到小角色
  - 发现初步线索
  - 获得小收获

Phase 3: 小冲突 (2-3章)
  - 与小反派/竞争者冲突
  - 展示主角当前实力
  - 小爽点

Phase 4: 受挫 (2-3章)
  - 遇到强大对手/困境
  - 计划失败
  - 盟友受伤/分散
  - 低谷点

Phase 5: 成长 (2-3章)
  - 获得新能力/新认知
  - 突破/升级
  - 找到新策略

Phase 6: 高潮 (2-4章)
  - 最终大战
  - 大打脸/大爽点
  - 解决本卷核心矛盾
  - 获得大奖励

Phase 7: 过渡 (1-2章)
  - 收获总结
  - 埋下一卷伏笔
  - 引出新目标
  - 章尾大钩子
```

## 16. 伏笔设计与回收

### 6种伏笔类型

| 类型 | 描述 | 回收距离 |
|------|------|----------|
| 物品伏笔 | 看似普通的物品后来有大用 | 10-100章 |
| 人物伏笔 | 路人甲后来是关键人物 | 20-200章 |
| 对话伏笔 | 某句话暗示未来事件 | 5-50章 |
| 场景伏笔 | 某处环境描写暗示秘密 | 10-50章 |
| 设定伏笔 | 世界规则的例外暗示真相 | 50-500章 |
| 情节伏笔 | 某个未解释的事件 | 20-100章 |

### 伏笔账本管理

```
伏笔生命周期:
  埋设 → 暗示(可选,多次) → 半揭示(可选) → 完全回收

管理规则:
  - 每卷至少埋3-5条新伏笔
  - 每卷至少回收1-2条旧伏笔
  - 跨卷回响: 早期伏笔在后期以更大规模回收
  - 回收时要有"恍然大悟"感
  - 避免: 埋了不收(遗忘) / 收得太突兀(强行)
```

## 角色设计方法论 (4层)

### 层级结构

```
Layer 1: 核心 (不变的本质)
  - 核心需求 (Need): 角色真正追求什么
  - 核心恐惧 (Fear): 角色最害怕什么
  - 核心信念 (Belief): 角色相信什么
  - Ghost/Wound: 过去的创伤

Layer 2: 外在 (可见的表现)
  - 目标 (Want): 角色以为自己想要什么
  - 性格标签: 3-5个显著特征
  - 行为模式: 压力下如何反应
  - 语言风格: 口头禅/用词习惯

Layer 3: 关系 (与他人的联系)
  - 对主角的态度和关系
  - 对其他角色的态度
  - 社会角色和地位
  - 信息差: 知道什么/不知道什么

Layer 4: 弧线 (变化轨迹)
  - 起点状态
  - 关键转变事件
  - 弧线方向: 正弧(成长)/负弧(堕落)/平弧(坚持)
  - 终点状态
```

## 额外框架

### Lester Dent 公式 (短篇/单元)
- 6000字分4个1500字块
- 每块必须有：新的冲突/线索/物理危险/转折
- 适合网文单章/短篇

### Marshall Plan (长篇)
- 完整的章节场景规划表
- 每个场景有：角色/目标/冲突/结果/牵涉角色列表

### W-Plot (多角色)
- 多条情节线交织形成W形
- 每条线有独立的上升/下降节奏
- 适合群像/多POV叙事
