# 六大项目实际使用流程对比

> 评估目标：了解每个项目从用户视角的完整操作流程

---

## 快速对比

| 项目 | 安装方式 | 开书到写第一章要几步 | 日更操作 | 自动化程度 |
|------|---------|-------------------|---------|-----------|
| **oh-story-claudecode** | `npx skills add` 一键 | 7步（setup→选题→设定→大纲→细纲→写作→检查） | 说"日更" | ⭐⭐⭐⭐ |
| **claude-novel-writeFlow** | clone 到项目 | 3步（/style → /outline → /write） | `/write` | ⭐⭐⭐ |
| **writeAgent** | clone + claude . | 3步（立项→大纲→写作循环） | "继续写下一章" | ⭐⭐⭐⭐ |
| **novelist-workbench** | clone + claude . | 4步（开书→5问→章数锁定→写） | "继续写下一章" | ⭐⭐⭐ |
| **awesome-novel-skill** | install.sh | 5步（项目→设定→章纲→提示词→写作） | "写第X章" | ⭐⭐⭐⭐ |
| **InkOS** | npm i -g @actalk/inkos | 2步（inkos book create → inkos write next） | `inkos write next` 或 `inkos up` 守护进程 | ⭐⭐⭐⭐⭐ |

## 详细流程

### 1. oh-story-claudecode (997 stars)

**安装：**
```
npx skills add worldwonderer/oh-story-claudecode -y
/story-setup
```

**完整流程：**
```
Step 1: /story-setup                    → 部署 hooks/agents/rules
Step 2: /story-long-scan (可选)          → 扫榜分析市场趋势
Step 3: /story-long-analyze (可选)       → 拆解对标小说
Step 4: /story-long-write               → Phase1 选题方向（用户回答）
Step 5: 回答设定问题                     → Phase2 生成 设定/角色/世界观
Step 6: 确认设定                        → Phase3 生成 大纲+卷纲+前30章细纲
Step 7: 确认大纲                        → Phase4 逐章写作+自动质检
Step 8: /story-deslop (按需)            → 六大门禁去AI味
```

**日更：** 说"日更"或"续写"，自动加载追踪文件→确认章节范围→逐章写作→自动更新伏笔/时间线

**产出目录：**
```
{书名}/
├── 设定/（角色/*.md + 世界观/*.md + 关系.md + 题材定位.md）
├── 大纲/（大纲.md + 卷纲_*.md + 细纲_第*章.md）
├── 正文/（第001章_*.md ...）
├── 追踪/（上下文.md + 伏笔.md + 时间线.md）
└── 对标/（拆文报告）
```

### 2. claude-novel-writeFlow

**安装：**
```
git clone https://github.com/jinmawang/claude-novel-writeFlow.git my-novel
cd my-novel && claude .
```

**完整流程：**
```
Step 1: /style                → 5轮问答定义风格 → style-rules.md (10维度)
Step 2: /outline --auto       → 5轮问答 → 4路并行头脑风暴 → 5方案选1 → outline/
Step 3: /write --auto         → 三智能体循环(Writer+StyleReviewer+ContinuityReviewer, 最多3轮) → chapters/
Step 4: /status              → 查看进度
Step 5: /export              → 合并导出
```

**日更：** `/write` 自动写下一章

**产出目录：**
```
my-novel/
├── style-rules.md
├── outline/（overview.md + chapter-01.md ...）
├── chapters/（chapter-01.md ...）
└── context/（characters.md + world.md + continuity.md + timeline.md）
```

### 3. writeAgent

**安装：**
```
git clone https://github.com/Ddhjx-code/writeAgent.git
cd writeAgent && claude .
```

**完整流程：**
```
Step 1: "我想写一本修仙小说"    → 主Agent引导立项，3个方向选1
Step 2: 确认方向               → planner生成 bible/(大纲+角色+世界观)
Step 3: 确认大纲               → 进入逐章循环（每章6步全自动）：
  3a. planner 规划场景          → plans/chNN-plan.md
  3b. writer 写正文             → chapters/chNN.md
  3c. reviewer 10维审查         → reviews/chNN-review.md (满分50, >=35通过)
  3d. 主Agent判断通过/修改/重写
  3e. polisher 润色去AI味       → 覆写 chapters/chNN.md
  3f. planner 归档更新bible     → 更新 bible/ 各文件
Step 4: 每5章里程碑审查         → 用户确认方向
```

**日更：** "继续写下一章"，自动进入6步循环

**产出目录：**
```
writeAgent/
├── bible/（plot/outline.md + characters/*.md + worldbuilding/*）
├── plans/（chNN-plan.md）
├── chapters/（chNN.md）
├── reviews/（chNN-review.md）
└── skills/（子Agent技能包）
```

### 4. novelist-workbench

**安装：**
```
git clone https://github.com/huahaiwujiang/novelist-workbench.git
cd web-novelist && claude .
```

**完整流程：**
```
Step 1: "使用chinese-novelist帮我写小说"  → 创建 novels/<书名>/ 目录
Step 2: 回答5问                          → 题材/主角设定/性格/冲突/章数
Step 3: 确认章数(锁定不可改)              → 生成 00-大纲.md + 01-人物档案.md
Step 4: 确认规划                          → 逐章全自动写作（每章自动）：
  - 写前规划 → 撰写(>=3000字) → 字数检查 → 连贯性检查
  → humanizer-zh去AI味(>=35/50) → 最终字数检查 → 生成摘要
```

**日更：** "继续写下一章，与前章串行衔接"

**产出目录：**
```
novels/<书名>/
├── 00-大纲.md（含每章摘要，逐章更新）
├── 01-人物档案.md
├── 第01章.md ...
└── (仿写时) 风格参考手册.md
```

### 5. awesome-novel-skill

**安装：**
```
git clone https://github.com/modoojunko/awesome-novel-skill.git
./install.sh claude-code
```

**完整流程：**
```
Step 1: "帮我写一本玄幻小说"        → Phase1+2: 创建项目+设定世界观角色
Step 2: 确认设定                    → Phase3: 规划章纲(分卷→分章)
Step 3: 确认章纲                    → Phase4: 自动拆segment+生成提示词 (自动)
Step 4: "写第X章"                  → Phase5: subagent写全章+质检+去AI味
Step 5: 写完                       → Phase6: 自动归档+更新状态 (自动)
```

**特色：**
- YAML状态机自动路由（outline→draft→archived）
- 24种题材预设
- 模型门禁（Phase1-4必须Sonnet，Phase5-6可用Haiku）
- 节奏监控（连续3+章高压 或 2+章平淡 自动报警）
- 两种模式：步步确认(默认) / 全部授权("你全权决定")

### 6. InkOS (6k stars)

**安装：**
```
npm i -g @actalk/inkos
inkos init my-novel
cd my-novel
inkos                    # 启动 Studio Web 工作台
```

**完整流程：**
```
Step 1: inkos book create --title "吞天魔帝" --genre xuanhuan
  → 生成 story_bible.md + book_rules.md + author_intent.md

Step 2: inkos write next 吞天魔帝    # 写一章（或 --count 5 写5章）
  → 10-Agent 全自动管线：
  Radar(扫趋势) → Planner(读意图) → Composer(编上下文) → Architect(规划结构)
  → Writer(写正文) → Observer(提取9类事实) → Reflector(JSON delta写入)
  → Normalizer(字数治理) → Auditor(33维审计) → Reviser(修复问题)
  → 不通过则循环 Reviser→Auditor 直到关键问题清零

Step 3: inkos review approve-all 吞天魔帝   # 审阅通过
Step 4: inkos export 吞天魔帝               # 导出
```

**守护进程（全自动日更）：**
```
inkos up        # 启动后台，自动循环写章
inkos down      # 停止
```
配合 Telegram/飞书 通知，关键问题暂停等人工审核。

**7个真相文件（每章自动更新）：**
- current_state.md（角色位置/关系/情绪）
- particle_ledger.md（物资账本）
- pending_hooks.md（伏笔追踪）
- chapter_summaries.md（章节摘要）
- subplot_board.md（支线进度）
- emotional_arcs.md（情感弧线）
- character_matrix.md（角色交互矩阵）

## 关键差异对比

### 记忆/一致性方案对比

| 项目 | 记忆方案 | 更新机制 | 伏笔追踪 |
|------|---------|---------|---------|
| oh-story | 追踪/3文件(上下文+伏笔+时间线) | 每章自动更新 | ✅ 有 |
| writeFlow | context/4文件(角色+世界+连续+时间线) | /write后增量提取 | ❌ 无专门 |
| writeAgent | bible/(大纲+角色+世界观) | 每章planner归档 | ❌ 无专门 |
| workbench | 00-大纲.md(含摘要) | 每章追加摘要 | ❌ 无 |
| awesome-novel | YAML状态+memo | 每章归档更新 | ✅ 有(钩子追踪) |
| **InkOS** | **7真相文件(JSON+MD)** | **Observer+Reflector自动** | **✅ pending_hooks.md** |

### 审查体系对比

| 项目 | 审查方式 | 维度 | 自动修复 |
|------|---------|------|---------|
| oh-story | 4Agent对抗审查 | 禁用词+钩子+伏笔 | ✅ |
| writeFlow | 三智能体(Style+Continuity) | 8+4+4+5+2=23项 | ✅ 最多3轮 |
| writeAgent | reviewer 10维度 | 50分制(>=35通过) | ✅ 判定后重写 |
| workbench | humanizer-zh | 去AI分(>=35/50) + 字数检查 | ✅ 低于2500字重写 |
| awesome-novel | 10维60+细项 | 双轮净化+AI味自检 | ✅ |
| **InkOS** | **Auditor 33维度** | **最全面** | **✅ Reviser循环** |

### 去AI味方案对比

| 项目 | 去AI味方法 |
|------|-----------|
| oh-story | **六大门禁**（禁用词→句式→心理外化→节奏打碎→对话去腔调→结尾去升华），分级轻/中/重 |
| writeFlow | style-rules.md 含AI味检测清单(8项机械感+4项风格) |
| writeAgent | polisher Agent 专职去AI味+语言精修+节奏微调 |
| workbench | **humanizer-zh** 24模式检测修正，评分制(>=35/50) |
| awesome-novel | 禁用句式/高频词/模板化身体反应/公式化副词/分析句式 |
| InkOS | 33维审计含AI痕迹检测，Writer规则内置去AI味 |
