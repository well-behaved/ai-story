# 学术论文与研究

AI长篇小说生成相关的核心论文、架构模式和评估方法。

---

## 核心论文详解

### Plan-and-Write (2019, ACL)

- **论文**: "Plan-And-Write: Towards Better Automatic Storytelling"
- **会议**: ACL 2019
- **核心贡献**: 首次系统性地提出规划与写作分离的范式
- **方法**:
  - 阶段1: 生成故事线(storyline) — 关键词序列
  - 阶段2: 基于故事线生成正文
  - 动态规划: 根据已生成内容调整后续规划
- **评估**: 人类评估显示规划后的文本在连贯性和趣味性上显著优于端到端生成
- **影响**: 几乎所有后续长篇生成工作都采用了"先规划后写作"范式

---

### PlotMachines (2020, EMNLP)

- **论文**: "PlotMachines: Outline-Conditioned Story Generation with Dynamic Plot State Tracking"
- **会议**: EMNLP 2020
- **核心贡献**: 动态情节状态追踪
- **方法**:
  - 输入: 故事大纲(outline)
  - 维护动态plot state: 追踪当前情节状态
  - 内存网络: 存储和检索情节信息
  - 条件生成: 基于大纲+当前状态生成下一段
- **关键洞察**: 需要显式追踪"故事进展到哪里了"
- **影响**: 后续的状态追踪方法(Story Bible/实体状态)可追溯到此

---

### DOC (2022, arXiv:2212.10077)

- **论文**: "DOC: Improving Long Story Coherence With Detailed Outline Control"
- **arXiv**: 2212.10077
- **核心贡献**: 详细大纲控制显著提升长文质量
- **量化结果**:
  - +22.5% 连贯性
  - +28.2% 大纲相关性
  - +20.7% 趣味性
- **方法**:
  - 多级大纲: 从粗到细的层次化大纲
  - 严格遵循: 生成时强制遵循大纲约束
  - 详细程度: 大纲越详细，生成质量越高
- **关键发现**: 大纲详细程度与生成质量正相关

---

### Re3 (2022, arXiv:2210.06774)

- **论文**: "Re3: Generating Longer Stories With Recursive Reprompting and Revision"
- **arXiv**: 2210.06774
- **核心贡献**: 递归重提示+修订+重排序
- **方法**:
  - **Recursive Reprompting**: 将长文生成分解为递归的短文生成
  - **Revision**: 每段生成后进行自我修订
  - **Reranking**: 多个候选中选择最优
  - 记忆管理: 维护故事计划(plan)+前文摘要(premise)
- **关键创新**: "递归"思想——将大问题分解为可处理的小问题
- **影响**: 验证了"多次生成+选择最优"策略的有效性

---

### Dramatron (2022, arXiv:2209.14958)

- **论文**: "Co-Writing Screenplays and Theatre Scripts with Language Models"
- **arXiv**: 2209.14958
- **作者**: DeepMind团队
- **核心贡献**: 分层提示链(hierarchical prompting)用于戏剧创作
- **方法**:
  - 分层生成链:
    1. Log line (一句话概念)
    2. Characters (角色设计)
    3. Plot beats (情节节拍)
    4. Scene descriptions (场景描述)
    5. Dialogue (对话)
  - 每层的输出作为下一层的输入约束
  - 用户可在每层介入修改
- **评估**: 15位业内专业人士(编剧/导演/戏剧家)评估
  - 整体认为有创意辅助价值
  - 主要问题: 过于通用/模板化
  - 建议: 人工参与越多质量越高
- **影响**: "分层生成"成为后续标准范式

---

### RecurrentGPT (2023, arXiv:2305.13304)

- **论文**: "RecurrentGPT: Interactive Generation of (Arbitrarily) Long Text"
- **arXiv**: 2305.13304
- **核心贡献**: 用自然语言模拟RNN的循环记忆
- **方法**:
  - 类比RNN架构:
    - Hidden State → 短期记忆(当前段落摘要)
    - Cell State → 长期记忆(全文摘要)
  - 每步输入: 上一步output + 短期记忆 + 长期记忆 + 规划
  - 每步输出: 新段落 + 更新的短期记忆 + 更新的规划
  - 理论上可无限延续
- **交互模式**: 
  - 自动模式: 完全自主续写
  - 交互模式: 人类选择下一步方向
- **影响**: 证明了纯提示工程也能实现长期记忆

---

### LongStory (2023, arXiv:2311.15208)

- **论文**: "LongStory: Coherent, Complete and Length Controlled Long Story Generation"
- **arXiv**: 2311.15208
- **核心贡献**: 两个关键技术解决长篇问题
- **方法**:
  - **CWC (Context Weight Calibration)**: 上下文权重校准
    - 问题: 随着文本变长，模型对早期内容关注度下降
    - 解决: 校准attention权重，确保关键早期信息不被遗忘
  - **LSP (Long Story Structure Position)**: 长篇故事结构位置
    - 问题: 模型不知道"故事进行到哪里了"
    - 解决: 注入结构位置信息(开头/发展/高潮/结尾)
- **影响**: 技术性解决attention衰减问题

---

### HoLLMwood (2024, arXiv:2406.11683)

- **论文**: "HoLLMwood: Unleashing the Creativity of Large Language Models in Screenwriting via Multi-Agent Collaboration"
- **arXiv**: 2406.11683
- **核心贡献**: 多Agent角色扮演用于剧本创作
- **方法**:
  - 多Agent协作:
    - Writer Agent: 编剧
    - Editor Agent: 编辑，提供修改建议
    - Actor Agents: 扮演角色，验证对话自然性
  - 角色扮演验证: Actor Agent从角色视角评估对话是否"符合人设"
  - 迭代修订: Editor反馈→Writer修改→Actor验证
- **关键创新**: 让AI"扮演"角色来验证角色一致性

---

### LongWriter (2024, arXiv:2408.07055)

- **论文**: "LongWriter: Unleashing 10,000+ Word Generation from Long Context LLMs"
- **arXiv**: 2408.07055
- **核心贡献**: 突破LLM单次生成长度限制
- **方法**:
  - **AgentWrite**: 任务分解方法
    - 将长文生成分解为多个子任务
    - 每个子任务生成一部分
    - 最后拼接为完整长文
  - **LongWriter-6k数据集**: 6000+超长文本训练样本
  - **LongBench-Write**: 长文生成评估基准
- **结果**: 使开源模型能单次生成10000+字
- **影响**: 解决了开源模型"输出长度瓶颈"

### StoryBox (AAAI 2026)

- **论文**: "StoryBox: Mixed-Initiative Long Story Generation"
- **会议**: AAAI 2026
- **核心贡献**: 混合自下而上的长篇生成方法
- **方法**:
  - 混合主动性(Mixed-Initiative): 人机协作
  - 自下而上: 从片段组装为完整故事
  - 结构化约束: 保证全局连贯

### ReasoningNCP (2025, arXiv:2503.22828)

- **论文**: 关于推理增强的叙事连贯性
- **arXiv**: 2503.22828
- **核心贡献**: RL奖励范式 VR-CLI
- **方法**:
  - VR-CLI: Value-based Reasoning for Creative Literature Innovation
  - 使用强化学习优化叙事连贯性
  - 奖励函数基于连贯性/一致性指标

### DOME (NAACL 2025)

- **论文**: "DOME: Dynamic Outline-Memory Enhanced Long Story Generation"
- **会议**: NAACL 2025
- **核心贡献**: 动态分层大纲 + 记忆增强
- **方法**:
  - 动态大纲: 大纲可以随生成过程调整
  - 记忆增强: 显式记忆模块存储关键信息
  - 分层: 全局大纲→章节大纲→场景大纲

## 5种架构模式

### 模式1: Hierarchical Generation (层次化生成)

```
代表论文: Dramatron, DOC, DOME
流程: Concept → Outline → Detailed Outline → Scenes → Text
特点: 从抽象到具体逐层展开
优势: 结构清晰、全局可控
局限: 一旦上层错误影响全局
```

### 模式2: Recursive/Iterative (递归/迭代)

```
代表论文: RecurrentGPT, Re3
流程: Generate → Summarize → Plan → Generate (循环)
特点: 每步更新记忆，循环推进
优势: 理论上可无限延续
局限: 信息逐步衰减、记忆管理复杂
```

### 模式3: Multi-Agent Collaboration (多Agent协作)

```
代表论文: HoLLMwood, ChatDev
流程: 多个Agent分工 + 互相审查
特点: 不同Agent负责不同维度
优势: 专业化分工、质量审查
局限: 通信成本高、一致性挑战
```

### 模式4: Plan-Write-Revise (规划-写作-修订)

```
代表论文: Plan-and-Write, Re3
流程: Plan → Write → Review → Revise → Plan(下一段)
特点: 写后修订、迭代改进
优势: 质量持续改进
局限: 成本高(多轮调用)
```

### 模式5: State-Tracking (状态追踪)

```
代表论文: PlotMachines, LongStory
流程: 维护显式状态 → 条件生成 → 更新状态
特点: 显式追踪故事状态
优势: 一致性保证
局限: 状态设计复杂
```

## 评估维度

### 7大评估维度

| 维度 | 定义 | 评估方法 |
|------|------|----------|
| 连贯性 (Coherence) | 句间/段间/章间逻辑连续 | 人类评分+NLI模型 |
| 一致性 (Consistency) | 角色/设定/时间线不矛盾 | 事实提取+对比 |
| 相关性 (Relevance) | 内容是否与大纲/主题相关 | 大纲对比评分 |
| 趣味性 (Interestingness) | 是否吸引人继续阅读 | 人类评分 |
| 流畅性 (Fluency) | 语言是否通顺自然 | 自动指标+人类 |
| 创意性 (Creativity) | 是否有新颖的表达和情节 | 人类评分 |
| 完整性 (Completeness) | 故事是否有完整的弧线 | 结构分析 |

### 评估方法

**自动评估**:
| 指标 | 测量 | 工具 |
|------|------|------|
| Perplexity | 流畅度 | GPT-2/LLaMA |
| BLEU/ROUGE | 与参考的相似度 | nltk/rouge-score |
| BERTScore | 语义相似度 | bert-score |
| Repetition-n | n-gram重复率 | 自定义 |
| Distinct-n | n-gram多样性 | 自定义 |
| Entity Consistency | 实体一致性 | NER+规则 |
| NLI Contradiction | 矛盾检测 | NLI模型 |

**人类评估**:
- A/B对比测试
- Likert量表打分(1-5或1-10)
- 维度分解评分
- 专家评估 vs 众包评估

**LLM-as-Judge**:
- 用GPT-4/Claude评估生成质量
- 与人类评分相关性: 0.7-0.85
- 成本低，可大规模使用
- 注意: 可能偏好自己的输出

## 角色一致性5种技术

### 1. Character Sheet Conditioning

```
方法: 将角色设定sheet作为持久上下文注入
实现: 每次生成时附加角色核心信息
效果: 基本一致性保证
局限: 上下文空间占用
```

### 2. Dialogue-Action Consistency Check

```
方法: 检查角色的对话和行为是否符合设定
实现: 
  - 提取对话中的态度/情感
  - 与角色设定对比
  - 标记不一致
效果: 发现显性矛盾
局限: 无法捕捉微妙的OOC
```

### 3. Role-Playing Verification (HoLLMwood)

```
方法: 让AI"扮演"该角色，判断是否会说出这句话
实现:
  - Actor Agent接收角色设定
  - 判断: "作为[角色]，我会说这句话吗？"
  - 给出Yes/No + 理由
效果: 捕捉微妙的不一致
局限: 成本高(额外调用)
```

### 4. Character Voice Fingerprinting

```
方法: 建立每个角色的"语言指纹"
实现:
  - 分析角色历史对话的特征(用词/句长/口头禅)
  - 生成时约束符合指纹
  - 检测偏离度
效果: 语言风格一致
局限: 需要足够的样本建立指纹
```

### 5. Knowledge Boundary Enforcement

```
方法: 严格执行"谁知道什么"
实现:
  - 维护每个角色的知识状态
  - 生成时检查角色是否"应该知道"这个信息
  - 防止全知视角污染
效果: 防止信息穿越
局限: 知识状态维护复杂
代表: Dramatica-Flow的信息边界系统
```

## 情节结构和故事弧生成方法

### 故事弧建模

```
数值化情感弧线:
  - 每个场景赋予情感强度值(1-10)
  - 绘制情感曲线
  - 与目标弧线(如Vonnegut的故事形状)对比
  - 调整节奏使实际曲线接近目标

常见弧线形状:
  - 男孩遇见女孩: ↑↓↑ (上升-下降-上升)
  - 灰姑娘: ↑↑↓↑↑ (持续上升-短暂挫折-最终胜利)
  - 悲剧: ↑↑↓↓ (上升后持续下降)
  - 起起落落: ↑↓↑↓↑ (反复波动上升)
```

### 情节生成策略

1. **骨架优先**: 先生成关键转折点，再填充中间
2. **因果链**: 每个事件必须是前一事件的因或果
3. **多线编织**: 多条情节线独立发展，在关键节点交汇
4. **张力曲线**: 维持张力持续上升，定期释放(微爽)保持阅读动力

## 值得关注的前沿方向

1. **超长上下文模型** (1M+ tokens): 可能根本改变记忆管理需求
2. **RL优化叙事质量**: 用强化学习直接优化"好看"指标
3. **多模态叙事**: 文本+图像+音频联合生成
4. **交互式叙事AI**: 读者选择影响故事走向
5. **文化感知生成**: 模型理解特定文化的叙事惯例
6. **可控生成**: 精确控制情节走向/情感/风格的同时保持创意
