## Top 3 推荐

### Top 1: oh-story-claudecode (997 stars) ★★★★★

与我们方案完美匹配。原生 Claude Code，13 Skills + 6 Agents + 5 Hooks，明确番茄/起点适配。

**实际架构：**
- 6 Agent：story-architect(Opus)、character-designer(Sonnet)、narrative-writer(Sonnet)、consistency-checker(Haiku)、story-researcher(Sonnet)、story-explorer(Haiku)
- 文件结构：设定/（世界观+角色+势力+关系）、大纲/（大纲+卷纲+细纲）、正文/、追踪/（上下文+伏笔+时间线）
- Hooks：compact前后进度快照和恢复，设定缺口检测
- 110+ 种技法知识库按需加载
- 日更续写：每3章自动快照，细纲缺失必须先补再写
- story-deslop 六大门禁去AI味

### Top 2: writeAgent ★★★★

文件记忆实现最接近我们目标。

**实际架构：**
- 4 子Agent：planner(规划/归档/bible维护)、writer(写作)、reviewer(10维审查)、polisher(去AI味)
- 文件记忆：bible/(人物+世界观+情节)、plans/、chapters/、reviews/、skills/
- 10维审查满分50，>=35通过
- 每5章里程碑审查

### Top 3: claude-novel-writeFlow ★★★★

实现最精巧。

**实际架构：**
- /outline --auto：4并行智能体头脑风暴（市场+人物+叙事+主题），汇聚5方案
- /write：三智能体(Writer+Style Reviewer+Continuity Reviewer)，最多3轮
- /context：维护4追踪文件，每章增量更新
- 上下文控制：overview截3000字、前章末尾800字、前前章300字

## 从各项目中"偷"什么

### 骨架层（← oh-story-claudecode）
- Skill 组织方式（每个 Skill 独立目录 + SKILL.md + references/）
- Agent 定义格式（.claude/agents/*.md）
- Hooks 机制（session-start、pre-compact、post-compact）
- setup 一键部署
- 日更续写场景匹配逻辑

### 记忆层（← InkOS + Write-Agent + claude-novel-writeFlow）
- 7 真相文件的 Markdown 格式（简化为 5 个核心文件）
- P0-P6 优先级组装策略（Token 预算内动态截断）
- 每章写完自动提取事实增量
- 伏笔紧迫度分级（短/中/长期 + 回收窗口）

### 写作层（← claude-novel-writeFlow + writeAgent）
- 三智能体写作循环（最多3轮）
- 10维审查评分体系
- 上下文加载策略（前章末尾800字+前前章300字+本章细纲+伏笔表）

### 去AI味层（← oh-story-claudecode + Claude-Book）
- 六大门禁清除法
- 困惑度改善器思路
- 分级处理（轻度/中度/重度）

### 番茄特化层（← Lorn.NovelWriteSkills + ai-novel-writer）
- 番茄平台输出模板
- Playwright 自动发布脚本
- BypassAIGC 降重
- 收入预估模型

## 8 个坑要避免

1. **上下文爆炸** → P0-P6 优先级截断，永远不全量加载
2. **Compact 后失忆** → pre/post-compact hooks 保存/恢复进度快照
3. **并行写章致一致性崩** → 强制串行逐章，细纲缺失不许动笔
4. **Bible 被意外修改** → 设定文件只读，变更走专门 planner Agent
5. **伏笔断线** → pending_hooks.md + 紧迫度分级 + 紧急回收强制提醒
6. **去AI味停留表面** → 六大门禁分层处理（不只换词，要打碎节奏）
7. **YAML状态增加复杂度** → 用 Markdown + 文件存在性标记代替
8. **过度工程化** → 控制 4-6 个 Agent，不引入额外框架依赖

---

## 最终推荐方案

**以 oh-story-claudecode 为骨架** → 强化记忆层（InkOS真相文件 + Write-Agent P0-P6 + writeFlow context增量） → 嫁接番茄特化（Lorn平台模板 + ai-novel-writer自动发布）

最终规模：**5-6 个 Skill + 4-5 个 Agent + 3-4 个 Hooks**
