# 工作流自动化平台

9个主要平台的详细分析和通用流水线架构模式。

---

## 1. Coze (扣子)

- **提供方**: ByteDance (字节跳动)
- **URL**: https://www.coze.com / https://www.coze.cn
- **定价**: 免费 (国内版有额度限制)

### 核心功能

- **Bot**: 创建AI机器人，定义人设/知识/技能
- **Workflow**: 可视化工作流编排
- **Loop节点**: 支持循环迭代(关键！用于逐章生成)
- **知识库**: 上传文档供Bot检索
- **插件**: 300+官方插件
- **发布**: 一键发布到多个平台

### 小说写作工作流特点

```
典型Coze写作工作流:
  开始 → 输入(题材/大纲) → LLM(生成章节大纲) 
  → Loop(逐章循环):
      → LLM(生成本章正文)
      → LLM(质量检查)
      → [通过?] → 下一章 / [不通过] → 重写
  → 汇总输出
```

### 社区生态

- 社区最大的AI写作工作流生态
- "星月AI小说工作流": B站99万播放
- 大量免费模板可直接使用
- 用户活跃度高

### 优缺点

- **优势**: 免费、中文生态最好、社区活跃、模板多
- **劣势**: 国际版和国内版不互通、复杂逻辑受限、上下文管理弱

## 2. Dify

- **提供方**: LangGenius (开源)
- **URL**: https://dify.ai
- **定价**: 开源免费 / 云版有付费tier
- **GitHub Stars**: 60k+

### 核心功能

- **工作流引擎**: 可视化编排，支持复杂逻辑
- **原生迭代节点**: 
  - 并行迭代: 多个任务同时执行
  - 串行迭代: 逐一执行
  - 最多10项并行
- **RAG知识库**: 内置向量检索，多种分块策略
- **变量系统**: 丰富的变量传递和转换
- **Agent模式**: ReAct/Function Call两种Agent模式
- **API发布**: 一键生成API接口

### 小说写作工作流特点

```
Dify写作工作流:
  输入(大纲JSON) 
  → 迭代节点(串行,每章):
      → 知识库检索(相关上下文)
      → LLM(组装上下文+生成)
      → 代码节点(提取事实更新记忆)
      → 条件判断(质量检查)
  → HTTP节点(保存结果)
```

### 优缺点

- **优势**: 开源、技术最成熟、迭代节点强大、RAG内建、可私有部署
- **劣势**: 迭代最多10项(需要分批)、UI学习曲线、部署需服务器

## 3. n8n

- **提供方**: n8n GmbH (开源)
- **URL**: https://n8n.io
- **定价**: 开源免费 / 云版$20+/月
- **GitHub Stars**: 50k+

### 核心功能

- **400+服务连接**: 几乎所有常用服务都有集成
- **SplitInBatches**: 批量处理节点(关键！逐章处理)
- **Code节点**: JavaScript/Python自定义逻辑
- **子工作流**: 工作流嵌套调用
- **Webhook**: HTTP触发
- **错误处理**: 完善的异常处理机制

### 小说写作工作流特点

```
n8n写作工作流:
  Webhook触发(接收写作任务)
  → 读取大纲(Google Sheets/Notion)
  → SplitInBatches(逐章处理):
      → HTTP Request(调用LLM API)
      → Code节点(后处理/一致性检查)
      → Google Docs/Notion(保存章节)
      → Wait(避免API限流)
  → Email通知(完成)
```

### 优缺点

- **优势**: 最灵活、400+集成、可连接任何服务、强大代码节点
- **劣势**: 需要自行搭建AI集成、无内建LLM/RAG、UI偏技术

---

## 4. FastGPT

- **提供方**: labring (开源)
- **URL**: https://fastgpt.in
- **定价**: 开源免费 / 商业版
- **GitHub Stars**: 20k+

### 核心功能

- **知识库最强**:
  - 多知识库混合检索
  - 精确匹配+语义检索混合
  - 多种分块策略(QA对/手动标注/自动分块)
  - 引用溯源
- **工作流**: 可视化编排
- **变量/分支**: 条件逻辑
- **HTTP调用**: 外部服务集成

### 小说写作适用场景

```
FastGPT最适合的场景:
  - 世界观知识库管理(上传设定文档)
  - 角色信息检索(精确匹配角色名)
  - 前文回顾(语义检索相关段落)
  - 规则约束(检索世界规则)

组合使用:
  FastGPT作为RAG知识库层
  + Dify/n8n作为工作流编排层
  + LLM API作为生成层
```

### 优缺点

- **优势**: 知识库功能最强、检索最准、支持多策略、中文优化好
- **劣势**: 工作流能力弱于Dify/n8n、写作专用功能需自建

## 5. Make.com (原Integromat)

- **提供方**: Make
- **URL**: https://www.make.com
- **定价**: 免费版(1000操作/月) / $9+/月

### 核心功能

- **Iterator模块**: 遍历数组(逐章处理)
- **1500+应用连接**: 全球SaaS连接最多
- **数据存储**: 内建简单数据库
- **错误处理**: 完善的异常路由
- **调度**: 定时触发

### 小说写作工作流特点

```
Make写作工作流:
  Schedule触发(每日写作)
  → Google Sheets(读取今日章节大纲)
  → Iterator(逐章):
      → OpenAI/Claude(生成正文)
      → 数据存储(保存结果)
      → Slack通知(进度)
  → Google Docs(汇总)
```

### 优缺点

- **优势**: 1500+应用连接、可视化清晰、稳定可靠
- **劣势**: 付费、操作数限制、国内访问不便、无内建AI能力

## 6. Zapier

- **提供方**: Zapier
- **URL**: https://zapier.com
- **定价**: 免费版(100任务/月) / $19.99+/月

### 核心功能

- **Conversation with memory**: AI对话可保持记忆
- **6000+应用连接**: 连接最多的平台
- **最简单**: 无需编程知识
- **Tables**: 内建数据库
- **Interfaces**: 简单前端

### 优缺点

- **优势**: 最简单易用、6000+应用、AI记忆功能
- **劣势**: 贵、复杂逻辑受限、循环能力弱、不适合大批量

## 7. ComfyUI LLM Party

- **提供方**: 社区开源
- **URL**: GitHub
- **定价**: 免费

### 核心功能

- **文本+图像联合流水线**: 文字生成+配图一体化
- **角色扮演模型**: 可设定AI人格
- **节点式编排**: ComfyUI的节点拖拽界面
- **多模态**: 文本/图像/语音

### 小说写作适用场景

```
ComfyUI LLM Party适合:
  - 文字+插图联合生成
  - 封面设计流水线
  - 角色立绘生成
  - 场景图生成
  - 有声书配音流水线(+语音节点)
```

### 优缺点

- **优势**: 多模态唯一解、图文联合、社区活跃
- **劣势**: 学习曲线陡峭、UI复杂、纯文本场景不如Dify

- **RAG流水线**: 快速搭建检索增强
- **向量数据库集成**: 多种向量库支持
- **API导出**: 一键生成API

### 小说写作适用场景

```
适合快速搭建:
  - RAG检索增强写作
  - 多Agent对话式写作
  - 知识库管理
```

### 优缺点

- **优势**: LangChain完整可视化、快速原型、RAG友好

## 通用流水线架构模式

### 模式一：简单循环

```
适合: 快速验证、短篇
工具: Coze / Dify

[输入大纲] → Loop(逐章) {
  [组装上下文] → [LLM生成] → [保存]
} → [输出]
```

### 模式二：生成-检查-修订

```
适合: 中等质量要求
工具: Dify / n8n

[输入大纲] → Loop(逐章) {
  [组装上下文] → [LLM生成] → [质量检查LLM] 
  → {通过 → [保存]} / {失败 → [修订LLM] → [重新检查]}
} → [输出]
```

### 模式三：多Agent流水线

```
适合: 高质量要求
工具: n8n + 代码 / 纯代码

[输入大纲] → Loop(逐章) {
  [RAG检索上下文]
  → [Writer Agent] → [章节草稿]
  → [Reviewer Agent] → [审查报告]
  → {通过?} → [Updater Agent(更新记忆)] → [保存]
  → {不通过?} → [Reviser Agent] → [回到Reviewer]
} → [全局检查] → [输出]
```

### 模式四：事件驱动

```
适合: 长期运行、持续连载
工具: n8n + 消息队列

[定时触发/手动触发]
  → [读取进度] → [确定今日任务]
  → [生成章节] → [检查] → [保存]
  → [通知作者审核]
  → [作者确认] → [发布到平台]
  → [更新进度]
```

## 平台选择建议

| 需求 | 推荐平台 | 理由 |
|------|----------|------|
| 快速免费开始 | Coze | 免费+中文生态+模板多 |
| 技术能力强+开源 | Dify | 功能最全+可私有部署 |
| 需要连接多服务 | n8n | 400+集成+灵活度最高 |
| 强RAG知识库 | FastGPT | 检索最准+中文优化 |
| 需要图文联合 | ComfyUI | 唯一的多模态方案 |
| 极简零代码 | Zapier/Make | 最简单但有限 |
| LangChain可视化 | Flowise/LangFlow | 快速原型 |

### 推荐组合

```
最佳组合方案:
  编排层: Dify (工作流+迭代) 或 n8n (更灵活)
  知识层: FastGPT (RAG) 或 Dify内建
  模型层: DeepSeek/Claude API
  存储层: Notion/Google Docs
  通知层: 飞书/Slack/Email
  前端层: Coze Bot (用户交互)
```
