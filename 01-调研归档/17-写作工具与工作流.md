## 2. Notion

### 数据库驱动结构

**Characters数据库**:
| 属性 | 类型 | 说明 |
|------|------|------|
| Name | Title | 角色名 |
| Role | Select | 主角/配角/反派 |
| Cultivation | Select | 修为等级 |
| Location | Relation→Locations | 当前位置 |
| Relationships | Relation→Characters | 关系 |
| Status | Select | 活跃/离场/死亡 |
| First Appearance | Number | 首次出现章节 |
| Voice Notes | Rich Text | 语言风格 |

**Locations数据库**:
| 属性 | 类型 | 说明 |
|------|------|------|
| Name | Title | 地点名 |
| Type | Select | 城市/宗门/秘境/野外 |
| Region | Select | 所属区域 |
| Characters | Relation→Characters | 在此的角色 |
| Events | Relation→Chapters | 相关章节 |

**Chapters数据库**:
| 属性 | 类型 | 说明 |
|------|------|------|
| Title | Title | 章节标题 |
| Number | Number | 编号 |
| Volume | Select | 所属卷 |
| Status | Select | 构思/初稿/修改/定稿 |
| Word Count | Number | 字数 |
| Characters | Relation→Characters | 出场角色 |
| Plot Threads | Relation→Plot Threads | 推进的情节线 |
| Cool Points | Multi-select | 爽点类型 |
| Hook | Select | 钩子类型 |

**Plot Threads数据库**:
| 属性 | 类型 | 说明 |
|------|------|------|
| Thread Name | Title | 情节线名 |
| Type | Select | 主线/支线/伏笔 |
| Status | Select | 活跃/暂停/已解决 |
| Planted | Number | 开始章节 |
| Expected Resolve | Number | 预计回收章节 |
| Related Chapters | Relation→Chapters | 相关章节 |

### Notion AI内置功能

- **续写**: 选中文本后AI续写
- **改善写作**: 优化语言表达
- **翻译**: 多语言翻译
- **摘要**: 自动生成摘要
- **头脑风暴**: 列出创意点子
- **解释**: 解释复杂段落

### 外部自动化(Make/Zapier)

```
自动化示例:
  触发: Notion中Chapter状态改为"定稿"
  → Make/Zapier检测变更
  → 调用AI API生成摘要
  → 更新Notion中的Chapter Summary字段
  → 更新Characters数据库中的相关角色状态
  → 发送通知
```

### 中文网文Notion模板

```
推荐数据库视图:
  1. 写作看板: 按Status分组的Kanban
  2. 时间线: 按章节号的Timeline视图
  3. 角色出场表: 角色×章节的Table
  4. 进度日历: Calendar视图追踪日更
  5. 数据面板: 字数/进度/完成度汇总
```

## 3. Scrivener

### 三联核心

**Binder (活页夹)**:
- 左侧目录树
- 无限层级
- 拖拽重排
- 文档+文件夹混合

**Corkboard (软木板)**:
- 卡片式概览
- 每章一张卡片
- 显示: 标题+简短摘要
- 适合结构规划

**Outliner (大纲)**:
- 表格式大纲
- 自定义列(字数/状态/POV/等)
- 全局概览所有章节属性

### Compile导出

```
Scrivener Compile功能:
  - 定义导出格式(ePub/MOBI/PDF/Word/纯文本)
  - 自定义样式映射
  - 分隔符控制
  - 前/后附录
  - 目录自动生成
  - 多套Compile预设(网文/出版/投稿)
```

### Snapshot版本

```
Snapshot功能:
  - 为任意文档拍"快照"
  - 保留历史版本
  - 可对比不同版本差异
  - 适合: 每次重大修改前拍快照
  - 类似Git的简化版本控制
```

### 与Obsidian混合工作流

```
推荐混合工作流:
  Obsidian: 世界观/角色/研究/灵感 (知识管理)
  Scrivener: 实际写作/编辑/导出 (写作工具)
  
  工作流:
  1. Obsidian中构建世界观和角色(永久知识库)
  2. 在Obsidian中做规划和头脑风暴
  3. 将大纲导入Scrivener
  4. 在Scrivener中写作(利用Binder/Compile)
  5. 定期从Scrivener同步更新回Obsidian
  6. 最终用Scrivener Compile导出
```

## 4. Logseq / Roam Research

### 大纲式写作

```
特点:
  - 大纲式(每个bullet是一个块)
  - 块引用(在任何地方引用任何块)
  - 双向链接
  - 日记驱动(每日笔记)

写作应用:
  - 日记中记录灵感和进展
  - 块引用构建复杂关系网
  - 标签聚合同类内容
  - 适合: 非线性思考/碎片化创作
  
局限:
  - 不适合长文档编辑
  - 导出能力弱
  - 大纲式不适合阅读连续文本
```

### AI块操作

```
在Logseq/Roam中:
  - 选择一个块 → AI扩展
  - 选择多个块 → AI总结
  - 输入提纲 → AI展开为详细块
  - 适合: 灵感发散阶段
```

## 5. Ulysses / iA Writer / Typora

### 极简主义写作

```
共同特点:
  - 专注写作体验
  - Markdown原生
  - 零干扰界面
  - 快速流畅

写作场景:
  - 不需要复杂管理时
  - 纯粹的文字输出阶段
  - 已经在其他工具中规划好了
  - 需要"心流"状态

与AI配合:
  - 外部AI生成 → 粘贴到编辑器修改
  - 编辑器写好 → 复制到AI审查
  - 不追求集成，追求分离和专注
```

## 6. "第二大脑" + AI混合工作流

### RAG on Story Notes

```
架构:
  Obsidian笔记 → 向量化 → 向量数据库 → RAG检索 → AI写作

实现:
  1. 所有故事笔记(世界观/角色/大纲/已写章节)存入Obsidian
  2. 使用Smart Connections或外部工具向量化
  3. 写作时自动检索相关笔记注入AI上下文
  4. AI基于检索到的知识生成内容

好处:
  - 知识库随写作增长
  - 检索自动化，无需手动组装上下文
  - 保持笔记和写作的统一
```

### AI Story Consultant模式

```
将AI作为"故事顾问"而非"代笔":
  
  人工: 写初稿
  AI: "这里角色的反应合理吗？考虑到他之前的经历..."
  
  人工: 设计情节转折
  AI: "这个转折的伏笔够吗？需要在前面增加暗示吗？"
  
  人工: 困在某个场景
  AI: "从[角色]的角度，TA此刻最在意什么？"

用法: 不让AI写，让AI提问/建议/检查
```

### 迭代精炼循环

```
Iteration 1: AI生成粗稿(1000字)
Iteration 2: 人工标注(好的保留/需改的标记)
Iteration 3: AI重写标记部分
Iteration 4: 人工润色全文
Iteration 5: AI做最终检查(一致性/AI味)

每轮人工参与度: 10% → 30% → 10% → 40% → 10%
最终质量: 远高于单次生成
```

### Git版本控制

```
将写作项目Git化:
  
  git init novel-project
  
  .gitignore:
    .obsidian/workspace.json
    .obsidian/cache/
    *.bak
  
  分支策略:
    main: 定稿章节
    draft: 当前写作
    experiment: 实验性修改
  
  提交规范:
    "ch15: complete first draft"
    "ch15: revise after review"
    "bible: update character states after ch15"
    "outline: adjust vol2 based on reader feedback"
  
  好处:
    - 完整历史(可回退任何版本)
    - 分支实验(不影响主线)
    - diff查看修改
    - 多设备同步(GitHub private repo)
```

## 跨工具通用模式(5个)

### 1. 分离规划与执行

```
规划工具: Obsidian/Notion (结构化/可检索)
执行工具: Scrivener/Ulysses (流畅写作)
AI工具: Claude/DeepSeek (生成/审查)

不要在同一个工具中做所有事情。
```

### 2. 状态驱动写作

```
每个章节有明确状态:
  构思 → 大纲 → 初稿 → AI审查 → 人工修改 → 定稿

状态转换有明确条件:
  大纲→初稿: 大纲已审核通过
  初稿→AI审查: 字数达标+初步通读
  AI审查→修改: 有审查报告
  修改→定稿: 人工满意
```

### 3. 渐进式丰富

```
第一遍: 骨架(事件序列)
第二遍: 血肉(描写/对话)
第三遍: 灵魂(情感/主题)
第四遍: 打磨(风格/节奏)

不追求一步到位，每遍专注一个维度。
```

### 4. 定期同步

```
每章完成后:
  □ 更新Story Bible
  □ 更新角色状态
  □ 更新进度追踪
  □ 记录新伏笔/回收旧伏笔
  □ 更新摘要

每周:
  □ 全局一致性检查
  □ 进度评估
  □ 大纲微调
```

### 5. 备份和版本

```
3-2-1备份规则:
  3份副本
  2种存储介质
  1份异地

版本策略:
  Git(首选) 或 
  Scrivener Snapshots 或 
  定时压缩备份

永远不要只有一份副本。
```

## 按作者类型推荐表

| 类型 | 推荐工具组合 | 理由 |
|------|-------------|------|
| 技术型(会编程) | Obsidian + 自建脚本 + Claude API | 最灵活，完全自定义 |
| 效率型(追求产出) | Notion + Make + Claude | 自动化程度高 |
| 文学型(追求质量) | Scrivener + Obsidian + Claude Projects | 专业写作工具 |
| 极简型(低干扰) | iA Writer + ChatGPT | 最少工具 |
| 团队型(多人协作) | Notion + Google Docs + AI | 协作能力 |
| 网文型(日更连载) | Obsidian + DeepSeek API + 自动脚本 | 高效稳定 |
