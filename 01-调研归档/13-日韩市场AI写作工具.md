# 日韩市场AI写作工具

日本和韩国市场的AI写作工具详解，以及与中国/西方的差异对比。

---

## 日本市场

### AI Novelist (AIのべりすと)

- **URL**: https://ai-novel.com
- **定价**: 免费版(有限制) / 月额制
- **训练数据**: 2TB日文语料(小说/轻小说/同人)

#### Character Book 标签激活系统

**@Commands完整说明**:

| 命令 | 功能 | 用法 |
|------|------|------|
| @character | 定义角色信息 | @character[名前]{设定文本} |
| @activate | 激活角色/设定 | @activate[キャラ名] |
| @deactivate | 停用角色/设定 | @deactivate[キャラ名] |
| @scene | 场景切换 | @scene{场景描述} |
| @mood | 设定氛围 | @mood{暗い/明るい/緊張} |
| @pov | 切换视角 | @pov[キャラ名] |

**标签激活机制**:
- 设定关键词标签(如角色名、地点名)
- 当正文中出现该标签时，自动注入相关设定
- 类似NovelAI的Lorebook机制
- 支持优先级和条件触发

#### 场景括号指令

```
使用方括号控制场景:
  [バトルシーン] → 切换到战斗场景模式
  [ロマンスシーン] → 切换到浪漫场景模式  
  [日常シーン] → 切换到日常场景模式
  [シリアス] → 严肃/认真语调
  [コメディ] → 喜剧语调
```

#### 格式密度控制

| 参数 | 控制内容 | 范围 |
|------|----------|------|
| 対話密度 | 对话占比 | 0-100% |
| 省略点密度 | 省略号(…)使用频率 | 低/中/高 |
| 括弧密度 | 括号使用频率(心理描写) | 低/中/高 |
| 改行密度 | 换行频率(影响节奏感) | 低/中/高 |
| 読点密度 | 逗号(、)使用频率 | 低/中/高 |

**日文写作特有**:
- 「」: 对话
- （）: 心理活动/独白
- ……: 省略/沉默
- ——: 强调/转折

#### 最大生成长度
- 最大192K字符（约64K日文字符 ≈ 8-10万字小说）

#### MOD微调系统
- 用户可基于特定风格文本微调
- 创建个人化的写作模型变体
- 社区分享MOD

#### 生成预设

| 预设名 | 风格特点 |
|--------|----------|
| Ghostwriter | 幽灵代笔，通用创意 |
| Purple Proses | 华丽修辞，文学性 |
| Creative | 高创意度，不可预测 |
| Balanced | 平衡质量和连贯 |
| Precise | 精确跟随，低随机 |
| 轻小説風 | 轻小说风格 |
| 純文学風 | 纯文学风格 |

### NovelAI Genji

- **提供方**: NovelAI (Anlatan)
- **定位**: 专门优化的日语模型

#### 技术特色

- **双语BPE分词器**: novelai-tokenizer
  - 同时处理英日混合文本
  - 日文字符效率高于通用分词器
  - 支持片假名/平假名/汉字高效编码

- **文风模仿**:
  - 学习特定作者/作品的写作风格
  - 轻小说/纯文学/网文等类型风格
  - 句式/用词/节奏模仿

- **图像生成联动**:
  - NovelAI的AI绘图功能
  - 可为小说生成角色立绘
  - 场景插图生成
  - 动漫风格

#### 与AI Novelist的区别

| 维度 | AI Novelist | NovelAI Genji |
|------|-------------|---------------|
| 训练数据 | 2TB日文专项 | 多语言+日文优化 |
| 激活系统 | @commands | Lorebook关键词 |
| 图像 | 无 | 内建AI绘图 |
| 隐私 | 一般 | 端到端加密 |
| 社区 | 日本本土 | 国际化 |

### BunCho (文鳥)

- **URL**: https://buncho.ai
- **定位**: 专注前期规划(プロット生成)
- **平台**: 移动端优先

#### 核心功能

- **プロット生成** (Plot生成): AI辅助情节规划
- **キャラ設定** (角色设定): 角色设计辅助
- **あらすじ** (梗概): 故事梗概生成
- **場面展開** (场景展开): 从梗概到场景

#### 特点

- 轻量级：专注前期，不做完整正文生成
- 移动端友好：手机即可操作
- 日本轻小说/网文类型预设
- 适合用于灵感激发和初步构思

## 韩国市场

### Naver HyperCLOVA

- **提供方**: Naver (韩国最大互联网公司)
- **状态**: 内部工具，不对外开放写作功能
- **能力**: 韩语理解最强的模型之一
- **写作应用**: 内部实验中，未公开产品化

### Kakao Entertainment

- **提供方**: Kakao (韩国第二大互联网公司)
- **模式**: IP流水线 (网文→漫画→电视剧→游戏)
- **AI应用**: 
  - 内部AI辅助翻译(网文多语言)
  - AI辅助漫画着色
  - 潜在的AI辅助创作(未公开)
- **平台**: Kakao Page (韩国最大网文平台)

### Wrtn (뤼튼)

- **URL**: https://wrtn.ai
- **定位**: 韩国通用AI助手(类似中国的文心一言)
- **写作功能**:
  - 创意写作辅助
  - 故事生成
  - 角色设计
  - 非专门的长篇工具
- **特点**: 韩语原生，免费使用

### 韩国市场特征

| 特征 | 说明 |
|------|------|
| 垂直整合 | Naver/Kakao控制整个IP链 |
| AI辅助定位 | 辅助人类作者，非替代 |
| 读者反感AI | 韩国读者对AI生成内容反感强烈 |
| 网文-漫画联动 | Webtoon是核心变现渠道 |
| 翻译需求 | 韩国网文出海(中/日/英)需要AI翻译 |
| 平台管控 | Naver/Kakao对AI内容态度谨慎 |

## 日本AI写作工具的独特价值

### 1. 格式密度控制

日本工具的"格式密度"概念在其他市场罕见:
- 可以精确控制对话/描写/心理的比例
- 日文的标点符号系统丰富(「」『』(())……——)
- 轻小说有独特的格式惯例

### 2. 角色激活机制

AI Novelist的@command系统比Lorebook更直觉:
- 显式激活/停用角色
- 不依赖关键词出现
- 适合场景切换频繁的写作

### 3. 轻小说适配

日本工具深度理解轻小说惯例:
- 短段落(1-3行)
- 大量对话
- 心理描写用括号
- 特有的叙事节奏
- 吐槽文化(ツッコミ)

### 4. 同人创作支持

日本二次创作文化使AI写作在同人领域特别活跃:
- 原作角色设定可直接导入
- 社区分享角色卡
- 创作门槛降低
- 法律环境相对宽松(二次创作文化)

## 对中文AI写作工具的启示

### 可借鉴的点

1. **格式密度控制**: 中文网文也有对话密度/描写密度需求
2. **场景括号指令**: 简单直觉的场景切换机制
3. **MOD微调**: 用户自定义风格的便捷方式
4. **角色激活/停用**: 比Lorebook更显式的角色管理

### 不同点(需注意)

1. **字数**: 日本轻小说短，中国网文长(差10倍以上)
2. **节奏**: 日本节奏慢，中国网文追求快节奏/爽点密集
3. **类型**: 日本偏日常/治愈/恋爱，中国偏升级/打脸/爽文
4. **格式**: 日文标点体系完全不同
5. **审核**: 日本宽松，中国严格

## 市场机会分析

### 日本市场
- AI Novelist已占据主导
- 仍有轻小说专项优化空间
- 同人创作市场巨大(Comic Market/Pixiv)
- VTuber相关创作新方向

### 韩国市场
- 几乎空白(没有成熟产品)
- 但读者抵制情绪强
- Webtoon脚本辅助可能是突破口
- AI翻译是确定性需求

### 跨市场机会
- 中→日/韩翻译适配
- 日本轻小说→中文网文风格转换
- 韩国Webtoon→中国漫改协同
