## 8大去AI味技术

### 1. 个人语料微调

**原理**: 用个人写作样本微调模型，使其输出风格趋近个人特征。

**方法**:
- LoRA (Low-Rank Adaptation): 低秩适配，4-bit量化
- QLoRA: 量化LoRA，显存更低
- 训练数据: 个人作品5万+字，越多越好
- 格式: instruction/input/output 三元组

**成本**:
- 训练: $5-20 (云GPU租用)
- 显存要求: 7B模型约16GB，13B约24GB
- 训练时间: 几小时到一天

**效果**:
- 可显著降低AI检测率
- 保持个人独特表达
- 缺点：训练数据不足时效果有限

**工具**:
- LLaMA-Factory: 最简单的微调工具
- Axolotl: 功能全面
- Unsloth: 速度最快(2x加速)

### 2. 风格迁移 (Few-shot)

**原理**: 提供参考文本样本，从中提取风格特征，指导AI模仿。

**参考文本量**: 5000-20000字最佳

**20+维度风格提取**:

| 维度 | 分析内容 | 示例 |
|------|----------|------|
| 平均句长 | 字符数/词数统计 | 平均18字/句 |
| 句长变化 | 标准差，交替模式 | 短-长-短节奏 |
| 对话比例 | 对话占总字数% | 40%对话 |
| 描写密度 | 描写段落频率 | 每5段有1段纯描写 |
| 段落长度 | 平均段落字数 | 平均80字/段 |
| 词汇偏好 | 高频词/独特用词 | 偏好"幽幽""缓缓" |
| 标点习惯 | 省略号/破折号频率 | 多用省略号少用感叹号 |
| 叙事视角 | 第几人称/距离感 | 有限第三人称，近距离 |
| 时态使用 | 过去式/现在式 | 过去式为主 |
| 修辞频率 | 比喻/拟人密度 | 平均每段1个比喻 |
| 感官分布 | 视觉/听觉/触觉比例 | 视觉60%听觉25%其他15% |
| 节奏模式 | 快节奏/慢节奏切换 | 动作快→心理慢→动作快 |
| 对话标签 | "说"的替代词使用 | 少用标签，靠动作+语气 |
| 开头模式 | 章节开头习惯 | 多以环境描写开头 |
| 结尾模式 | 章节结尾习惯 | 多以悬念/反问结尾 |
| 过渡方式 | 场景转换手法 | 空行+时间标记 |
| 情感表达 | 直接/隐晦 | 隐晦为主，靠动作暗示 |
| 信息密度 | 每段新信息量 | 中等，留有呼吸空间 |
| 幽默方式 | 冷笑话/吐槽/荒诞 | 偶尔冷吐槽 |
| 文化引用 | 典故/流行文化 | 偶尔引用诗词 |

**提示模板**:
```
你是一位资深的小说作家。请分析以下参考文本的写作风格，并在后续写作中完全模仿这种风格。

参考文本:
"""
[5000-20000字参考文本]
"""

风格分析要求:
1. 句式特征(长短句交替规律)
2. 词汇偏好(高频用词/回避用词)
3. 对话风格(标签使用/语气特征)
4. 描写密度和方式
5. 节奏控制方式
6. 情感表达方式
7. 独特的表达习惯

请基于以上风格，用完全相同的风格写作以下内容:
[写作指令]
```

### 3. 多轮生成 + 人工混合

**原理**: 分多个Pass生成，逐步精修，最终混入人工修改。

**三轮生成法**:

```
Pass 1 - 骨架 (AI生成):
  目标: 完成情节和对话骨架
  参数: temperature=0.7, 注重逻辑性
  产出: 70%完成度的草稿

Pass 2 - 扩展 (AI生成):
  目标: 添加描写/心理/细节
  参数: temperature=0.9, 注重丰富性
  产出: 90%完成度的草稿

Pass 3 - 润色 (AI生成):
  目标: 风格统一/句式打磨/去重复
  参数: temperature=0.5, 注重精确性
  产出: 95%完成度

人工混合 (5-20%人工修改):
  - 修改AI味重的表达
  - 添加个人化表达
  - 调整节奏
  - 确保一致性
  - 加入独特创意
```

### 4. 突发性工程 (Burstiness Engineering)

**原理**: AI文本的一个显著特征是句长均匀(低突发性)，通过刻意制造句长变化来对抗检测。

**技术**:
- 3-5词超短句: "他停住了。" "不可能。" "血。"
- 30+词长句: 复杂从句/修饰堆叠
- 单词段落: 独立成段的单词增加视觉节奏
- 句子碎片: 不完整的句子，模拟思维碎片

**实现**:
```
提示指令:
"请在写作中刻意交替使用:
 - 极短句(3-5字): 至少每段1-2个
 - 中等句(10-20字): 主体
 - 长句(30+字): 偶尔使用复杂句
 - 单独成段的短语: 每200字至少1次
 
 避免所有句子长度相近。"
```

**示例**:
```
普通AI输出: "他走进了那间古老的图书馆，灰尘弥漫在空气中，让人不禁打了个喷嚏。书架上摆满了年代久远的书籍，散发着陈旧的气息。"

突发性改写: "图书馆。灰尘。他推开门的瞬间，扬起的微粒在斜射的光线里跳了一支无声的舞——三百年的寂静被打碎了。他打了个喷嚏。很响。那些书不会在意。"
```

### 5. 困惑度注入 (Perplexity Injection)

**原理**: AI检测器通过检测低困惑度(高可预测性)来识别AI文本，通过注入不可预测元素来提高困惑度。

**方法**:

**替换高频AI词**:
```
替换清单(中文):
  "然而" → "可偏偏" / "谁知" / 直接省略
  "不禁" → 去掉，改用动作
  "此刻" → "这会儿" / "当下"
  "仿佛" → "像是" / "好似" / 具体比喻
  "缓缓" → 具体动词
  "目光" → "眼神" / "视线" / 具体描写
  "嘴角微微上扬" → "笑了"
  "内心深处" → 去掉，直接写想法
```

**temperature调高**:
```
正文生成: temperature=0.9-1.2
  - 0.9: 轻微不可预测
  - 1.0: 中等随机
  - 1.2: 较高创意(可能需要人工修正)
```

**非标准标点使用**:
```
- 破折号代替逗号: "他转身——没看到身后的眼睛"
- 省略号碎片: "你说...那个人...不对，你根本没见过他"
- 感叹号+问号: "什么?!"
- 不加引号的对话: 他说走吧。她没动。
```

### 6. 语域混合 (Register Mixing)

**原理**: AI倾向使用统一的语域(正式/非正式)，混合不同语域增加自然感。

**混合策略**:
```
基础叙事: 书面语(中性)
角色对话: 口语/方言/网络用语
心理活动: 碎片化/口语化
描写段落: 文学性书面语
动作场景: 简短/节奏感

混合示例:
  [书面] 夕阳将天际染成了绯红，青云宗的钟声悠悠传来。
  [口语] "靠，又迟到了。"林轩拔腿就跑。
  [碎片心理] 完了完了完了。师兄要骂死他。
  [文学描写] 风穿过回廊，带走了最后一丝暖意。
  [网络语感] 这操作也太秀了，简直离谱。
```

### 7. 结构不可预测性

**原理**: AI倾向生成结构均匀的文本(每章差不多长、每段差不多结构)，打破这种均匀性。

**方法**:
```
章节长度差异:
  - 正常章节: 3000-4000字
  - 偶尔超短章: 800-1000字 (高强度场景)
  - 偶尔超长章: 6000+ (高潮章)
  
非时序叙事:
  - 偶尔闪回
  - 插入"三天前"/"一个月后"
  - 不同角色视角穿插
  
跑题段落:
  - 角色内心独白偏离主线
  - 环境描写超出需要
  - 看似无关的细节(其实是伏笔)
  
格式变化:
  - 插入信件/日记/告示
  - 纯对话章节(无叙事)
  - 独白章节
```

---

### 8. 刻意不完美 (Deliberate Imperfection)

**原理**: AI文本过于"完美"(无重复/无赘言/逻辑通顺)，加入人类写作中自然出现的"不完美"。

**方法**:
```
偶尔重复:
  - 角色口头禅重复使用
  - 关键词在段落中复现
  - "他说。他又说。" (非正式风格)

自我纠正:
  - "不对，不是这样——应该说..." (内心独白)
  - "等等，我想起来了"

冗余表达:
  - 同一件事从不同角度说两遍
  - 过渡句(实际信息量为零但增加节奏感)
  
不完美的比喻:
  - 勉强的类比
  - 角色自己觉得不恰当的比喻: "就像...算了，不是这样的"

断句/打断:
  - 说到一半被打断
  - 想法没说完就转移
```

## AI检测原理

### 核心指标

| 指标 | 含义 | AI文本特征 | 人类文本特征 |
|------|------|-----------|-------------|
| Perplexity (困惑度) | Token预测困难度 | 低(可预测) | 高(不可预测) |
| Burstiness (突发性) | 困惑度变化幅度 | 低(均匀) | 高(波动大) |
| Token概率 | 各Token被选中的概率 | 集中在高概率token | 分散 |
| 文体均匀性 | 风格是否一致 | 极度一致 | 有自然波动 |

### 检测方法分类

**统计方法**:
- 困惑度分析(大概率词占比)
- 突发性分析(句长/词复杂度变化)
- 零宽度特征(token概率分布)

**模型方法**:
- 训练分类器(人类vs AI)
- 水印检测(OpenAI水印)
- 风格分析模型

- 方法: 学术环境专用
- 准确率: 较高
- 中文支持: 中等

```

  1. 纯AI输出直接发布

```

## 自检测反馈循环

### 架构

```
生成 → 自检测 → [AI味评分] → 决策:
  score < 0.3: 通过，发布
  0.3 ≤ score < 0.6: 局部修改
  score ≥ 0.6: 重写

自检测方法:
  1. 调用检测API(GPTZero/自建)
  2. 计算困惑度(用小模型如GPT-2打分)
  3. 检查突发性(句长方差)
  4. 检查禁用词出现率
  5. 综合评分
```

### 代码示例

```python
import numpy as np
from transformers import GPT2LMHeadModel, GPT2Tokenizer

class AIDetectorSelfCheck:
    def __init__(self):
        # 用小模型计算困惑度
        self.model = GPT2LMHeadModel.from_pretrained("uer/gpt2-chinese-cluecorpussmall")
        self.tokenizer = GPT2Tokenizer.from_pretrained("uer/gpt2-chinese-cluecorpussmall")
        
        # AI禁用词表
        self.ai_words = ["然而", "不禁", "此刻", "仿佛", "缓缓", "目光", 
                         "内心深处", "嘴角微微", "轻声说道", "点了点头"]
    
    def calculate_perplexity(self, text):
        """计算文本困惑度 - 越低越像AI"""
        inputs = self.tokenizer(text, return_tensors="pt")
        with torch.no_grad():
            outputs = self.model(**inputs, labels=inputs["input_ids"])
        return torch.exp(outputs.loss).item()
    
    def calculate_burstiness(self, text):
        """计算突发性 - 越低越像AI"""
        sentences = text.split("。")
        lengths = [len(s) for s in sentences if s.strip()]
        if len(lengths) < 2:
            return 0
        return np.std(lengths) / np.mean(lengths)  # 变异系数
    
    def check_ai_words(self, text):
        """检查AI高频词出现率"""
        count = sum(text.count(word) for word in self.ai_words)
        return count / len(text) * 1000  # 每千字出现次数
    
    def evaluate(self, text):
        """综合评分: 0(完全人类) - 1(明显AI)"""
        perplexity = self.calculate_perplexity(text)
        burstiness = self.calculate_burstiness(text)
        ai_word_rate = self.check_ai_words(text)
        
        # 困惑度评分(低困惑度 = 高AI概率)
        ppl_score = max(0, 1 - perplexity / 100)
        
        # 突发性评分(低突发性 = 高AI概率)
        burst_score = max(0, 1 - burstiness / 0.8)
        
        # AI词评分
        word_score = min(1, ai_word_rate / 5)
        
        # 综合(加权平均)
        final_score = ppl_score * 0.4 + burst_score * 0.3 + word_score * 0.3
        
        return {
            "ai_probability": final_score,
            "perplexity": perplexity,
            "burstiness": burstiness,
            "ai_word_rate": ai_word_rate,
            "suggestion": self._get_suggestion(final_score)
        }
    
    def _get_suggestion(self, score):
        if score < 0.3:
            return "通过：AI痕迹较低"
        elif score < 0.6:
            return "建议：增加句长变化、替换AI高频词、加入口语化表达"
        else:
            return "需要重写：AI痕迹明显，建议降低temperature重新生成或大量人工修改"
```
