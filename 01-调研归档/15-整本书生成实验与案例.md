已知的整本书AI生成实验、成果、教训和最佳实践。

## 案例1: Tim Boucher — 97本AI小册子

### 基本信息
- **作者**: Tim Boucher (加拿大艺术家/作家)
- **时间**: 2023年
- **产出**: 97本AI生成小册子
- **字数**: 每本2,000-15,000字
- **用时**: 每本3-8小时
- **收入**: 初期约$2,000

### 方法

```
Tim Boucher的工作流:
1. 人工构思概念和主题
2. AI(ChatGPT)生成初稿
3. AI(Midjourney)生成封面和插图
4. 人工编辑和修改(约30%内容)
5. 自出版到Amazon KDP
6. 每本定价$1.99-3.99
```

### 关键发现

- 短篇(2k-5k字)质量最好，越长越难
- 世界观设定类(lore books)比叙事类成功
- 封面和书名对销量影响巨大
- 量变不一定带来质变——97本中只有少数有持续销售
- Amazon后来限制了每日发布数量

### 教训

- AI擅长短篇概念性内容，不擅长长篇叙事
- 人工策划+AI执行的模式可行
- 量产策略有天花板
- 读者最终看中的是独特性和质量

## 案例2: NaNoWriMo实验

### 基本信息
- **活动**: National Novel Writing Month (每年11月)
- **目标**: 一个月写50,000字小说
- **AI参与**: 2023-2024年大量作者使用AI辅助

### 典型AI辅助流程

```
NaNoWriMo AI辅助经验:
  时间分配:
    - 60%+时间在规划和编辑(非生成)
    - 20-40%内容最终被人工重写
    - 纯AI生成的部分约40-60%

  实际工作流:
    Week 1: 规划(人工90%+AI辅助10%)
      - 确定概念/角色/世界观
      - AI帮助头脑风暴和扩展
    Week 2-3: 写作(AI生成60%+人工40%)
      - AI生成初稿章节
      - 人工修改/重写/插入
    Week 4: 修改(人工80%+AI20%)
      - 大量人工编辑
      - AI帮助查找矛盾
```

### 关键数据

- 使用AI可将日均产出从1667字提升到3000-5000字
- 完成率(达到50k字)从~15%提升到~35%
- 但完成≠质量：AI辅助的50k字通常需要2-3倍编辑时间

### 社区反响

- 2023年NaNoWriMo官方对AI态度引发巨大争议
- 传统作家社区强烈反对
- "辅助"vs"生成"的边界定义不清
- 最终NaNoWriMo修改了规则，允许"AI辅助"但要求声明

## 案例3: Sudowrite Story Engine

### 基本信息
- **产品**: Sudowrite Story Engine
- **能力**: 从概念生成50,000-100,000字完整小说
- **技术**: "Story DNA"压缩上下文

### Story DNA技术

```
Story DNA的核心思想:
  问题: 小说太长，无法全部放入上下文
  解决: 将小说"压缩"为DNA——核心特征的结构化表示

  Story DNA包含:
  - 核心前提(1句)
  - 角色DNA(每个角色的核心驱动力)
  - 情节DNA(关键转折点序列)
  - 风格DNA(语调/节奏/视角特征)
  - 世界DNA(核心规则)

  生成时:
  Story DNA + 当前章节大纲 + 前几章摘要 → 新章节
```

### 实际效果

- 可以生成50k-100k字的完整故事
- 全局连贯性约70%（仍有矛盾）
- 需要大量人工引导（每章都需要确认/修改方向）
- 文学质量中等，适合类型小说
- 成本约$50-200/本（多次调用API）

### 用户反馈

- 适合作为"极粗糙的初稿"起点
- 不适合直接出版
- 规划阶段的价值 > 生成阶段
- 最适合：有经验的作者加速流程，而非新手替代写作

## 案例4: Amazon KDP洪水

### 现象
- 2023年起，AI生成书籍大量涌入Amazon KDP
- 2024年高峰期，每天数千本AI书上架
- 非虚构(self-help/how-to)比虚构更多

### Amazon的应对

```
Amazon KDP限制措施:
  - 限制每作者每天最多3本新书
  - 要求声明AI使用情况
  - 加强内容审核
  - 打击低质量批量上架
  - 标记AI生成内容(部分市场)
```

### 数据分析

- **非虚构 > 虚构**: AI生成的非虚构(指南/教程/食谱)销量更好
- **短篇 > 长篇**: 短篇电子书比长篇更适合AI
- **封面重要**: 好封面+好标题决定了90%的销量
- **评价两极**: 平均评分低于人工作品
- **退货率高**: AI小说退货率高于平均

### 成功模式

```
KDP上AI书籍的成功特征:
  1. 细分赛道(非常具体的niche)
  2. 高质量封面(不是AI默认审美)
  3. 人工深度编辑(30%+修改)
  4. SEO优化(标题/关键词/分类)
  5. 系列化(建立品牌/读者群)
  6. 非虚构 > 虚构(知识类更容易)
```

## 案例5: 中文网文AI生成

### 基本情况
- **目标字数**: 50-200万字
- **平台**: 番茄/七猫(免费)/起点(付费)
- **类型**: 玄幻/都市/系统文最多

### 分层生成法(中文网文)

```
中文网文AI生成主流方法:

Layer 1: 总纲 (人工50%+AI50%)
  - 确定类型/金手指/核心冲突
  - 规划分卷(10-20卷)
  - 确定主要角色(5-10人)
  - 字数: 2000-5000字

Layer 2: 分卷纲 (人工30%+AI70%)
  - 每卷目标和核心事件
  - 卷内角色发展
  - 升级节点分配
  - 字数: 每卷500-1000字

Layer 3: 章纲 (AI90%+人工审核10%)
  - 每章的核心事件/出场角色/爽点
  - 批量生成(一次50-100章)
  - 人工审核和调整
  - 字数: 每章50-100字

Layer 4: 正文 (AI95%+人工5%)
  - 逐章生成3000-4000字
  - 注入上下文(前文+设定+大纲)
  - 自动或人工审查
  - 字数: 3000-4000字/章

Layer 5: 一致性检查 (AI)
  - 检查角色/设定矛盾
  - 检查时间线
  - 检查修为等级是否合理
  - 标记问题让人工修正
```

### 实际数据

| 指标 | 典型值 |
|------|--------|
| 日产字数 | 10,000-30,000 |
| 人工参与度 | 5-30% |
| 一致性问题率 | 每10章1-3处 |
| 读者留存 | 低于纯人工作品20-40% |
| 封号风险 | 番茄低/起点高 |
| 月收入(中位数) | 1000-3000元 |

## 案例6: 递归摘要流水线

### 方法
- **目标**: 生成80,000-120,000字完整小说
- **核心**: Running Summary + 角色状态 + 情节追踪

### 具体实现

```
递归摘要流水线:

维护3个持续更新的文档:
  1. Running Summary (2000字): 全书到目前的摘要
  2. Character States (1000字): 所有角色当前状态
  3. Plot Tracker (500字): 活跃情节线+伏笔

每章生成流程:
  输入 = [Running Summary] + [Character States] + [Plot Tracker] 
        + [最近2-3章全文] + [当前章大纲]
  
  → LLM生成本章(3000-4000字)
  
  生成后:
  → LLM更新Running Summary(加入本章事件)
  → LLM更新Character States(角色变化)
  → LLM更新Plot Tracker(情节推进)
  → 保存，进入下一章
```

### 效果

- 可达80k-120k字
- 短期连贯性好(相邻章节)
- 长期可能出现"主题漂移"
- 早期伏笔容易丢失(被压缩掉)
- 需要人工定期审查Running Summary质量

## 8个关键质量问题

| # | 问题 | 严重度 | 出现频率 | 主要原因 |
|---|------|--------|----------|----------|
| 1 | 角色行为不一致(OOC) | 高 | 常见 | 上下文丢失/角色设定未注入 |
| 2 | 情节重复/循环 | 高 | 常见 | 模型倾向重复成功模式 |
| 3 | AI味重/文风机械 | 中高 | 很常见 | 模型默认输出风格 |
| 4 | 伏笔遗忘 | 高 | 常见 | 记忆管理不足 |
| 5 | 设定矛盾 | 中 | 偶尔 | Bible未及时更新 |
| 6 | 节奏单调 | 中 | 常见 | 缺少节奏变化指导 |
| 7 | 深度不足 | 中高 | 很常见 | AI难以自发生成深度 |
| 8 | 结局仓促 | 高 | 常见 | 长篇控制力不足 |

## "什么实际可行"的6条共识

基于所有案例的共识:

### 1. AI规划 > AI生成
规划阶段AI的价值最大。AI擅长：头脑风暴/结构设计/信息扩展。

### 2. 人机协作是必须的
0%人工参与=不可接受的质量。至少需要15-30%人工干预。

### 3. 短篇比长篇容易
5k字: AI几乎可以独立完成
50k字: 需要大量人工协调
200k+字: 极具挑战，需要完善的系统

### 4. 类型小说比文学创作适合
网文/通俗小说/类型小说：AI处理得更好(套路可循)
纯文学/实验性写作：AI几乎无法胜任

### 5. 系统化 > 随性写
有完善系统(Bible/RAG/审查)的项目 >>> 随意使用ChatGPT

### 6. 40%规划 / 20%生成 / 40%编辑
最佳时间分配不是"让AI写"，而是"做好前后工作"

## 坦诚评估

### 目前确实可行的

1. AI辅助规划(大纲/角色/世界观) — 效果极好
2. AI生成初稿+人工深度修改 — 可行且高效
3. 短篇/单元式内容批量生产 — 商业可行
4. 网文免费平台批量发布 — 有收入但风险大
5. AI做对话扩展/描写丰富 — 效果好
6. AI做一致性检查 — 非常有用

### 目前仍然困难的

1. 纯AI生成高质量长篇(>10万字) — 仍需人工
2. AI生成有深度的文学作品 — 几乎不可能
3. AI保持200k+字的完美一致性 — 需要复杂系统
4. AI生成让读者真正感动的内容 — 偶尔能做到但不稳定
5. 无人工参与的全自动出版流程 — 质量不达标
6. AI理解"什么才是好故事" — 缺乏真正的审美判断

## 最佳实践比例

```
理想的时间/精力分配:

■■■■■■■■ 40% 规划
  - 构思概念(人工主导,AI辅助)
  - 设计世界观(人工+AI)
  - 设计角色(人工为主)
  - 编写大纲(AI生成+人工修改)
  - 风格定义(分析参考+规范化)

■■■■ 20% 生成
  - AI逐章生成正文
  - 上下文组装和管理
  - 多版本生成和选择

■■■■■■■■ 40% 编辑
  - 一致性检查和修正
  - 去AI味处理
  - 风格打磨
  - 深度修改(加入人性化元素)
  - 全局审查(伏笔/弧线)
```
