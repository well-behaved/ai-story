## 分层提示链架构 (5个Chain)

### Chain 1: 概念生成

```
System: 你是一位经验丰富的网文策划编辑。

User: 请为以下类型生成5个核心概念，每个包含:
- 一句话高概念
- 核心金手指
- 主角设定(一句话)
- 核心冲突
- 独特卖点

类型: {genre}
目标读者: {audience}
参考作品: {references}
避免: {avoid_list}
```

### Chain 2: 世界观与角色

```
System: 你是一位资深的世界观设计师。基于以下核心概念进行详细设计。

User: 
核心概念: {concept_from_chain1}

请设计:
1. 世界观框架(力量体系/地理/社会结构)
2. 主角完整设定(4层: 核心/外在/关系/弧线)
3. 3-5位关键配角设定
4. 反派体系设计
5. 势力关系图

输出格式: YAML
```

### Chain 3: 大纲生成

```
System: 你是一位精通网文结构的大纲规划师。

User:
世界观: {world_from_chain2}
角色: {characters_from_chain2}
目标字数: {total_words}
目标章数: {total_chapters}

请生成四级大纲:
1. 总纲(全书主线+分卷规划)
2. 卷纲(每卷: 目标/核心事件/角色发展/字数)
3. 节纲(每5-10章: 小弧线结构)
4. 章纲(每章: 核心事件/出场角色/爽点/钩子)

确保:
- 每卷有完整的副本结构
- 伏笔合理分布(每卷埋3-5/收1-2)
- 爽点密度符合网文节奏
- 升级节奏合理
```

### Chain 4: 风格定义

```
System: 你是一位文体分析专家。

User:
参考文本(5000-20000字): {reference_text}

请分析并输出风格规范文档:
1. 句式特征(平均句长/变化规律)
2. 词汇偏好(高频词/禁用词)
3. 对话风格(标签习惯/语气词)
4. 描写方式(感官分布/比喻密度)
5. 节奏控制(快慢切换规律)
6. 特殊标点使用习惯
7. 段落长度分布
8. 每个角色的语音特征

输出为可直接注入到写作提示的规范文档。
```

### Chain 5: 章节生成

```
System: 你是一位{genre}类型的资深作者。严格遵循以下规范写作。

风格规范: {style_from_chain4}
Story Bible: {bible_core}
当前章大纲: {chapter_outline}
前文摘要: {previous_summary}
前一章结尾: {last_paragraph}
出场角色状态: {character_states}
活跃伏笔: {active_foreshadowing}

User: 请写第{N}章的正文。

要求:
1. 严格遵循章节大纲
2. 与前文无矛盾
3. 字数: {target_words}字
4. 必须包含: {required_elements}
5. 章尾必须有钩子
6. 避免以下AI味词汇: {banned_words}
```

## Dramatron方法详解

### 5层生成链

```
Layer 1 → Log Line (一句话故事核心)
  输入: 类型+主题+核心冲突
  输出: "当[主角]面对[触发事件]，必须[核心行动]，否则[赌注]"

Layer 2 → Characters (角色设计)
  输入: Log Line
  输出: 每个角色的名字/背景/动机/弧线/关系

Layer 3 → Plot Beats (情节节拍)
  输入: Log Line + Characters
  输出: 按时间顺序的关键情节点列表

Layer 4 → Scene Descriptions (场景描述)
  输入: Plot Beats + Characters
  输出: 每个场景的详细描述(地点/时间/参与角色/目标/冲突)

Layer 5 → Dialogue/Text (对话/正文)
  输入: Scene Description + Character Profiles
  输出: 完整正文
```

### 每层的人工干预点

- Layer 1: 选择/修改核心概念
- Layer 2: 调整角色设计
- Layer 3: 重排/增删情节节拍
- Layer 4: 细化场景要求
- Layer 5: 修改/重写具体段落

## 雪花法AI适配 (7步)

```
Step 1: 输入一句话概念
  Human: "修仙少年发现血脉秘密逆天改命"
  AI: 扩展为标准格式

Step 2: AI扩展为5句话
  AI: 设置句 + 灾难1 + 灾难2 + 灾难3 + 结局句

Step 3: AI生成角色卡(每人1页)
  AI: 名字/目标/动机/冲突/顿悟/一句话弧线

Step 4: AI将5句扩展为5段
  AI: 每句 → 完整段落

Step 5: AI为每个角色写完整故事线
  AI: 每个角色的完整经历(从他们的视角)

Step 6: AI扩展为4-5页大纲
  AI: 完整的故事大纲

Step 7: AI生成场景表
  AI: 每个场景一行(POV/地点/事件/结果)
```

## AI禁用词表

### 中文版

```
高频AI味词汇(必须避免):
  然而, 不禁, 此刻, 仿佛, 缓缓, 目光, 内心深处, 
  嘴角微微上扬, 轻声说道, 点了点头, 眼中闪过一丝,
  心中暗道, 不由得, 旋即, 须臾, 霎时间, 竟然,
  一股暖流, 莫名的, 淡淡的笑容, 若有所思,
  显然, 毫无疑问, 事实上, 值得注意的是

过度使用的结构(降低频率):
  "XX了XX" (点了点头/摇了摇头/叹了口气)
  "一丝XX" (一丝笑意/一丝寒意/一丝不安)
  "不由自主地" 
  "仿佛...一般"
  "似乎感觉到了什么"
  "xxx的目光中带着xxx"

建议替代:
  然而 → 可/偏偏/谁想到/直接省略
  不禁 → 删除，改用具体动作
  缓缓 → 具体动词(踱步/挪动/放下)
  目光 → 眼睛/视线/瞥/盯/瞪
  仿佛 → 像/好似/活脱脱/简直
  内心深处 → 删除，直接写想法
```

### 英文版

```
High-frequency AI words (must avoid):
  However, Moreover, Furthermore, Nevertheless, 
  Delve, Tapestry, Nuanced, Multifaceted,
  It's worth noting, It's important to note,
  In conclusion, That being said,
  A sense of, The weight of,
  His/Her eyes widened, A smile played on lips,
  Couldn't help but, Found himself/herself

Overused structures:
  "The [noun] of [noun]" (The weight of silence)
  "[verb]ed with [emotion]" (said with determination)
  "A [emotion] washed over" 
  "Little did [subject] know"
  "[Subject] couldn't help but [verb]"

Replacements:
  However → But/Yet/Still/Though/[delete]
  Moreover → And/Also/Plus/[delete]
  Delve → Dig/Explore/Look into
  Tapestry → [use concrete descriptions]
  Nuanced → Subtle/Complex/Layered
```

## Show Don't Tell 强制规则

```
提示注入:

规则: 绝对禁止直接说明情感，必须通过以下方式展示:
1. 身体反应: "拳头攥紧" 而非 "他很愤怒"
2. 动作细节: "摔门而去" 而非 "他生气地离开"
3. 感官描写: "太阳穴突突跳" 而非 "他很紧张"
4. 对话暗示: "你走吧" 而非 描述"她很伤心地说"
5. 环境映射: "乌云压城" 而非 "气氛很压抑"

禁止直接使用的情感词:
  高兴/悲伤/愤怒/恐惧/惊讶/厌恶/焦虑/兴奋
  → 全部替换为具体的身体/行为/感官描写

示例:
  ❌ "林轩非常愤怒"
  ✅ "林轩的指甲嵌入掌心，鲜血顺着指缝滴落，他浑然不觉"
  
  ❌ "苏灵很紧张"  
  ✅ "苏灵的手指无意识地绞着袖口的丝线，直到布料皱成一团"
```

## 多轮生成策略

### Pass 1: 骨架

```
System: 你是第一轮写手，负责情节骨架。

要求:
- 只写核心情节推进和关键对话
- 不需要详细描写
- 重点: 因果逻辑、角色行为合理性
- 字数: 目标字数的30-40%
- temperature: 0.7

输出标记需要扩展的位置:
  [EXPAND: 打斗场景，需要详细动作描写]
  [EXPAND: 环境描写，渲染紧张氛围]
  [EXPAND: 心理活动，展示内心挣扎]
```

### Pass 2: 扩展

```
System: 你是第二轮写手，负责填充血肉。

输入: Pass 1的骨架(含[EXPAND]标记)

要求:
- 将[EXPAND]标记位置展开为详细文本
- 添加感官描写、环境细节、心理活动
- 字数: 目标字数的90-100%
- temperature: 0.9
- 风格要求: {style_spec}
```

### Pass 3: 润色

```
System: 你是第三轮编辑，负责最终润色。

输入: Pass 2的完整文本

要求:
- 统一风格(检查风格偏离)
- 消除AI味词汇(参考禁用词表)
- 调整节奏(短句和长句的分布)
- 确保章尾钩子够强
- 检查连续性(与前文是否矛盾)
- temperature: 0.5
- 不要改变情节内容，只改表达方式
```

## AI写作常见失败模式及对抗提示

### 1. 紫色散文 (Purple Prose)

```
失败: 过度华丽的描写，每句都是比喻
对抗提示: 
  "描写密度控制: 每3段最多1段纯描写。
   比喻频率: 每500字最多2个。
   优先使用具体动词而非形容词堆砌。"
```

### 2. 角色同质化

```
失败: 所有角色说话方式相同
对抗提示:
  "角色语音表:
   林轩: 短句/少话/动作多/偶尔自嘲
   苏灵: 温柔/完整句/敬语/偶尔坚定
   王刚: 粗犷/口头禅'他娘的'/大嗓门/直接
   严格按此区分，不可混淆。"
```

### 3. 叙事惯性 (一直顺着写)

```
失败: 情节总是平滑推进，没有意外
对抗提示:
  "本章必须包含至少1个微型反转:
   - 读者以为会发生A，实际发生B
   - 角色的计划必须遇到意外阻碍
   - 禁止一切按计划顺利执行"
```

### 4. 总结代替展示

```
失败: "他们花了三天准备" (跳过有趣的细节)
对抗提示:
  "禁止时间跳跃超过1小时。
   所有事件必须场景化展示。
   如果需要时间推进，用场景切换而非叙述跳过。"
```

### 5. 无限扩写 (水字数)

```
失败: 无意义的描写和重复填充
对抗提示:
  "每段必须推进至少一个以下维度:
   □ 情节前进 □ 角色发展 □ 世界观展示 □ 情感深化 □ 伏笔暗示
   不推进任何维度的段落必须删除。"
```

### 6. 过度解释

```
失败: 解释每个角色的动机和想法
对抗提示:
  "信息分发规则:
   - 通过行为暗示，不直接说明
   - 读者不需要知道所有角色在想什么
   - 保留神秘感: 某些行为的动机留到后文揭示
   - 禁止 '他之所以这样做是因为...' 句式"
```
