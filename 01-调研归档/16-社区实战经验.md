# 社区实战经验

知乎/小红书/Reddit/HN等平台从业者的实战经验、工具推荐和收入数据。

## 3类从业者画像

### Type A: AI辅助作者

**画像**:
- 有一定写作经验（写过5万+字以上）
- 人做创意方向和关键决策
- AI写初稿，人工重写30-60%
- 日产5,000-15,000字
- 追求质量，在付费平台发表

**工作流**:
```
典型Type A工作流:
  早上(1h): 规划今日章节，写章纲
  上午(2-3h): AI生成2-3章初稿 + 实时修改
  下午(2h): 人工深度修改/润色
  晚上(1h): 审查一致性 + 准备明日
  
  总计: 4-7小时/天
  产出: 5000-15000字/天(净输出)
  平台: 起点/晋江(高质量要求)
```

**收入**: 3000-30000元/月（取决于平台和数据）

---

### Type B: AI流水线操作者

**画像**:
- 可能没有写作经验
- 高度自动化，人工参与<15%
- 在番茄/七猫等免费平台发表
- 日产10,000-30,000字
- 追求量，用多号分散风险
- 收入500-5000元/月
- 高封号风险

**工作流**:
```
典型Type B工作流:
  准备(一次性): 设定模板/工作流/世界观模板
  每日: 
    - 运行自动化流水线
    - 简单审核(10-30分钟)
    - 发布到平台
    - 监控数据
  
  总计: 1-2小时/天
  产出: 10000-30000字/天
  平台: 番茄/七猫/飞卢
  风险: 封号/降权/读者差评
```

**收入**: 500-5000元/月（不稳定，有封号归零风险）

---

### Type C: 严肃小说家(AI增强)

**画像**:
- 专业作家/签约作者
- AI做头脑风暴、查找矛盾、加速特定环节
- 重度重写(70%+人工)
- 使用Claude Projects管理长篇
- 追求出版级质量

**工作流**:
```
典型Type C工作流:
  构思阶段: Claude头脑风暴 → 人工选择和发展
  规划阶段: AI辅助大纲 → 人工深度修改
  写作阶段: 人工写核心场景 / AI帮助扩展次要场景
  修改阶段: AI做一致性检查/查找矛盾/建议修改
  
  AI参与度: 20-30%(主要在辅助环节)
  人工主导: 创意/决策/核心写作
  平台: 传统出版/起点精品/晋江
```

**收入**: 10000-100000+元/月（签约作者/头部收入）

## 上下文管理实战方案

### 社区常用方法

**方法1: Claude Projects**
```
在Claude Projects中:
  - 上传Story Bible(YAML/MD)到项目知识
  - 上传前5章全文作为风格参考
  - 每次对话注入: 当前章大纲+前章摘要+角色状态
  - 优势: 简单直接，200k上下文够用
  - 劣势: 手动管理，无自动化
```

**方法2: Obsidian + AI**
```
在Obsidian中:
  - 建立完整Fiction Vault
  - 使用Dataview追踪角色/场景/伏笔
  - Text Generator插件调用AI
  - Smart Connections检索相关笔记
  - 优势: 本地管理，结构清晰
  - 劣势: 上下文注入需手动
```

**方法3: 自建脚本**
```
Python/Node脚本:
  - 自动组装上下文(Bible+摘要+大纲)
  - 调用LLM API生成
  - 自动更新摘要和状态
  - 保存到文件系统
  - 优势: 完全自动化
  - 劣势: 需要技术能力
```

## 去AI味实战技巧(社区经验)

### 立竿见影的方法(知乎高赞)

```
1. 替换禁用词(效果: ★★★★★)
   - 准备替换词表，生成后批量替换
   - "然而"→删除 / "不禁"→删除 / "目光"→具体描写

2. 打乱句式(效果: ★★★★☆)
   - AI生成后手动打乱句子顺序
   - 长句拆短/短句合并
   - 加入碎片化短句

3. 加入口语/方言(效果: ★★★★☆)
   - 对话中加入方言/网络用语
   - 心理活动用口语
   - "我靠""得了""算了吧"

4. 混合使用多个模型(效果: ★★★☆☆)
   - Claude写一段→DeepSeek续一段→Qwen写一段
   - 不同模型的"味"不同，混合后更自然
   
5. 降temperature后手动加料(效果: ★★★★☆)
   - temperature=0.5生成稳定文本
   - 手动加入意外表达/独特比喻/个人化句子
```

### 成本排序(从低到高)

1. 批量替换禁用词 — 0成本，5分钟/章
2. 多模型混合 — 几乎无额外成本
3. 人工插入独特表达 — 每章10-15分钟
4. 风格微调模型 — $5-20一次性成本
5. 专人润色 — 每章30-60分钟

## 一致性维护实战方法

### 社区验证有效的方法

```
方法1: "写前检查清单"
  每章开始前强制AI回答:
  - 当前主角在哪里？
  - 身边有谁？
  - 最近发生了什么？
  - 有什么未解决的问题？
  → 如果答错，说明上下文不足

方法2: "章后事实提取"
  每章结束后让AI提取:
  - 本章新增了什么事实？
  - 哪些角色状态变了？
  - 有没有新伏笔？
  → 追加到Bible中

方法3: "定期全局审查"
  每10章让AI做一次全局检查:
  - 列出所有未回收的伏笔
  - 检查角色修为是否合理
  - 检查时间线有无矛盾
  - 检查有无重复情节

方法4: "角色日记法"(Reddit推荐)
  为每个主要角色维护一份"日记":
  - 记录他们的所见所闻
  - 记录他们的情感变化
  - 写作时从角色日记中提取信息
```

## 主要陷阱

### 1. 平台封号
```
风险因素:
  - 多账号操作被关联
  - AI特征被算法识别
  - 读者举报"像AI写的"
  - 短时间内发布过多作品
  
应对:
  - 加大人工修改比例(>30%)
  - 每个账号独立设备/IP
  - 建立独特的个人风格
  - 控制发布频率
```

### 2. 质量瓶颈
```
问题: AI生成内容在某个质量水平上无法突破
原因: 模型能力上限 + 上下文管理不足
表现: 读者流失/追读下降/差评增加
应对: 
  - 提高人工参与度
  - 使用更好的模型
  - 完善上下文管理系统
```

### 3. 重复模式
```
问题: AI倾向重复相似的情节和表达
表现: 每次冲突的解决方式类似/对话模式重复
应对:
  - 在大纲中明确要求多样性
  - 使用"禁止重复"检测
  - 定期更换提示模板
  - 多模型混合使用
```

### 4. 角色漂移
```
问题: 角色性格逐渐偏离初始设定
原因: 长期写作中角色设定被稀释
应对:
  - 每章注入角色核心设定
  - 定期"角色一致性审查"
  - 角色语音表始终在上下文中
```

### 5. 情节连贯
```
问题: 跨章节/跨卷的情节线断裂
原因: 记忆管理不足/摘要丢失信息
应对:
  - 伏笔账本独立维护
  - 情节线状态追踪
  - 定期全局审查
```

### 6. 成本失控
```
问题: API调用成本超过收入
实际数据:
  - 生成10万字约需: $5-50(取决于模型)
  - 多轮审查: 成本×2-3
  - 如果收入<3000元，成本需严格控制
应对:
  - 使用DeepSeek/Qwen等低成本API
  - 关键环节用好模型，批量用便宜模型
  - 本地部署降成本
```

### 7. 法律灰区
```
现状:
  - 中国目前无明确AI创作版权法规
  - AI生成内容的著作权归属不明确
  - 平台协议可能禁止AI内容
  - 未来法规方向不确定
注意:
  - 保留人工参与证据
  - 注意平台最新规定
  - 人工修改比例越高越安全
```

## 社区共识最佳工作流(5步)

```
Step 1: 选题和规划 (人工主导)
  - 研究市场(扫榜/看数据)
  - 确定类型和卖点
  - AI辅助头脑风暴多个方向
  - 人工选择最佳方向
  - 生成详细大纲(四级)

Step 2: 准备系统 (一次性)
  - 建立Story Bible
  - 设计角色卡
  - 定义风格规范
  - 建立上下文管理机制
  - 测试生成质量

Step 3: 日常写作循环
  - 组装上下文(Bible+摘要+大纲+前文)
  - AI生成当日章节(2-3章)
  - 人工审查和修改(30%+)
  - 更新记忆/摘要/状态
  - 发布

Step 4: 定期维护 (每周)
  - 全局一致性检查
  - 伏笔回收确认
  - 读者反馈分析
  - 大纲调整(根据数据)
  - 去AI味检查

Step 5: 运营优化
  - 监控平台数据
  - A/B测试不同写法
  - 优化成本
  - 维护读者关系
```

## 新兴趋势(2025)

### 1. AI检测加强
- 起点/晋江加强AI检测
- 读者鉴AI能力提升
- "像AI"成为差评理由
- 需要更高人工参与度

### 2. 从"AI生成"转向"AI辅助"
- 纯AI生成→AI辅助创作
- 工具定位从"替代"到"加速"
- 人机协作模式成为主流
- "AI增强型作者"身份认同

### 3. 多模型流水线
- 不再依赖单一模型
- Claude做创意+DeepSeek做批量+Qwen做中文优化
- 模型组合 > 单一最强模型

### 4. 本地模型降成本
- Qwen2.5-7B/14B质量已可用
- 批量生成+去AI味用本地模型
- 关键审核才用商业API
- 成本从$50/10万字→$5/10万字

### 5. 语音克隆
- AI有声书(TTS)成为新增收入渠道
- 配合小说生成有声版
- 多角色语音
- 平台开始接受AI有声内容
