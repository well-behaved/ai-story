# Story Start - 立项

## 前置检查

1. 确认 `.story-deployed` 存在（否则引导到 story-setup）
2. 确认 `.story-state` 不存在（否则提示已有项目）

## 模式

- **交互模式**（默认）：逐一询问 5 问，每步等用户确认
- **快速启动模式**：用户一次性提供设定信息，跳过逐步确认。触发方式：
  - 用户说"快速开书：系统文，男主18岁学生，逆袭变强，万界商城系统，轻松搞笑"
  - 或用户说"自动模式开书"
- 快速启动时，从用户输入中提取 5 问答案，生成所有设定后只做一次总确认
- 快速启动时，Step 7 的 .story-state 中 `auto_mode` 设为 `true`（后续 story-outline 自动跳过审核暂停）

## 流程

### Step 1: 5问收集

用 AskUserQuestion 逐一询问（不要一次全问，逐步引导）：

**Q1 题材类型**
- 选项：系统文 / 升级文 / 重生文 / 都市文 / 赘婿文 / 末日文 / 诡异复苏 / 无限流 / 其他
- 如果用户选"其他"，追问具体类型

**Q2 主角设定**
- 性别（男/女）
- 年龄段（少年/青年/中年）
- 初始处境（落魄/被欺/穿越/重生/觉醒/继承）

**Q3 核心冲突**
- 选项：变强复仇 / 拯救世界 / 逆袭称霸 / 守护至亲 / 探索真相 / 其他
- 追问具体细节

**Q4 金手指/系统**
- 主角的特殊能力是什么？
- 如果是系统文：系统的核心功能是什么？
- 如果不是系统文：主角的独特优势是什么？

**Q5 基调**
- 选项：轻松搞笑 / 热血燃向 / 阴暗复仇 / 稳重发展 / 爽快无脑 / 其他

### Step 1.5: 加载题材模板

根据 Q1 选择的题材，检查 `references/genre-templates/` 下是否有对应模板：
- 系统文 → `references/genre-templates/系统文.md`
- 升级文 → `references/genre-templates/升级文.md`
- 重生文 → `references/genre-templates/重生文.md`
- 都市文 → `references/genre-templates/都市文.md`
- 赘婿文 → `references/genre-templates/赘婿文.md`

如果存在，加载该模板作为后续 Step 2-5 的参考输入（世界观框架、角色配置、情节结构、爽点类型、金手指建议、番茄注意事项）。

如果用户选了"其他"题材且无对应模板，跳过此步。

### Step 2: 生成书名与简介

基于5问回答 + 题材模板（如有），生成：
- 3个书名方案（让用户选择或自定义）
- 一句话简介（20字以内）
- 详细简介（200字左右，用于平台发布）
- 卖点分析（为什么读者会看）

用户确认后，创建目录并写入：

```
{书名}/
├── 00-设定/
│   └── 书名与简介.md
```

### Step 3: 生成世界观与力量体系

调用 Planner Agent，基于题材和设定生成：

- **世界观**：地理概况、主要势力（3-5个）、历史背景、核心规则
- **力量体系**：7-12个大等级、每级名称和特征、突破条件、主角当前等级

写入：
```
{书名}/00-设定/世界观.md
{书名}/00-设定/力量体系.md
```

用户审核修改后继续。

### Step 4: 生成角色

调用 Planner Agent，生成主角和2-3个核心配角：

每个角色使用4层模板：
```markdown
# {角色名}

## 表层
- 外貌：{简要描写}
- 年龄：{数字}
- 口癖/习惯动作：{具体行为}

## 性格层
- 核心特质：{2-3个}
- 致命缺陷：{1个}
- 价值观：{一句话}

## 动机层
- 最大欲望：{具体目标}
- 最大恐惧：{具体恐惧}
- 核心秘密：{如有}

## 弧光层
- 起始状态：{故事开始时的状态}
- 终极状态：{故事结束时应变成的样子}
- 变化轨迹：{简要描述变化过程}

## 当前状态
- 境界：{等级}
- 位置：{地点}
- 关系：{与其他角色的关系}
- 持有物：{关键物品}
```

写入 `{书名}/01-角色/` 目录。

### Step 5: 生成风格规范

基于题材和用户偏好生成：

```markdown
# 风格规范

## 叙事视角
{第三人称限定/第一人称/全知}，主视角角色：{主角名}

## 句式偏好
- 短句为主（10-15字），偶尔长句（25+字）
- 段落 3-5 行为主
- 对话占比 40-60%

## 语言风格
- {轻松幽默/冷峻硬朗/热血激昂/...}
- 避免文艺腔，口语化表达

## 禁用词（50+）
不禁、缓缓、竟然、值得注意的是、不由得、忍不住、心中暗道、
目光一凝、嘴角微扬、深吸一口气、一股暖流、时间仿佛、
总的来说、显然、此外、然而（频繁使用时）、
眼神暗了暗、嘴角勾起一抹弧度、
心中涌起一股XX的感觉、不禁陷入沉思...
（完整列表在 references/anti-ai-rules.md）

## 禁用模式
- 连续3句相同句式开头
- "他感到X"直接描述情绪
- 章末哲理总结/人生感悟
- 开头回顾前文
```

写入 `{书名}/00-设定/风格规范.md`。

### Step 6: 初始化追踪文件

创建 5 个空模板追踪文件 + 归档目录：

```
{书名}/03-追踪/角色状态.md    ← 从角色档案初始化
{书名}/03-追踪/世界状态.md    ← 从世界观初始化
{书名}/03-追踪/伏笔账本.md    ← 空模板
{书名}/03-追踪/章节摘要.md    ← 空
{书名}/03-追踪/情节线.md      ← 空模板（主线+预留副线）
{书名}/03-追踪/归档/          ← 空目录（用于后续卷切换时归档旧数据）
{书名}/05-审查/               ← 空目录（用于审查报告持久化）
```

### Step 7: 初始化 .story-state

在项目根目录写入：

```json
{
  "book_name": "{书名}",
  "book_dir": "{书名}",
  "genre": "{题材}",
  "platform": "fanqie",
  "current_volume": 0,
  "current_chapter": 0,
  "chapters_planned": 0,
  "total_words": 0,
  "recent_scores": [],
  "auto_mode": false,
  "created_at": "{日期}",
  "status": "initialized"
}
```

**字段说明**：
- `current_volume`: 初始为 0（未开始），story-outline 生成第一卷后更新为 1
- `book_dir`: 项目子目录名，story-write 依赖此字段定位文件
- `status`: 状态机 initialized → outlined → writing → completed
- `recent_scores`: 最近 10 章审查分数，用于连续低分检测（Phase 3）
- `auto_mode`: false 交互模式（默认），true 跳过确认暂停

### Step 8: 完成提示

```
✅ 《{书名}》项目已创建

已生成：
  - 设定：世界观 + 力量体系 + 风格规范
  - 角色：{主角} + {配角1} + {配角2}
  - 追踪文件：已初始化

下一步：说"写大纲"生成总纲和章纲
```
