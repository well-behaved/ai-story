# Story Review - 全局审查

## 触发条件

1. **用户手动**：说"全局审查"或"检查一致性"
2. **卷末建议**：story-write 写完一卷最后一章时建议执行
3. **自动触发**：story-write 检测到 recent_scores 连续 3 章 < 40 分

## 前置检查

1. `.story-state` 存在且 `status` 为 "writing"
2. `current_chapter` >= 10（至少写了 10 章才有审查意义）

## 审查范围

- 默认：当前卷所有已写章节
- 用户可指定："审查第 1-50 章"、"审查第二卷"、"全书审查"

## 6 维全局审查

### 审查 1：角色性格偏移

```
输入：
  - 01-角色/ 下所有角色的原始设定（4层模板中的"性格层"+"动机层"）
  - 03-追踪/角色状态.md（当前状态）
  - 04-正文/ 中最近 10 章的角色对话和行为

检查：
  - 角色核心特质是否在最近章节中体现？
  - 是否出现与设定矛盾的行为？（如设定"冷静"但频繁冲动）
  - 角色说话方式是否漂移？（如口癖消失、语气变化）
  - 角色之间的关系演变是否合理？

输出：
  每个角色一行评分：
  - {角色名}：一致性 {高/中/低}，{具体问题描述}
```

### 审查 2：力量体系自洽

```
输入：
  - 00-设定/力量体系.md
  - 03-追踪/角色状态.md（各角色当前等级）
  - 最近 20 章中涉及战斗/突破的章节摘要

检查：
  - 主角的等级提升是否符合设定的突破条件？
  - 是否有越级战斗违反了设定的规则？
  - 不同角色之间的实力对比是否一致？
  - 技能/招式使用是否与当前等级匹配？

输出：
  - 自洽性评级：{高/中/低}
  - 具体矛盾列表
```

### 审查 3：时间线一致

```
输入：
  - 03-追踪/章节摘要.md（全部摘要）
  - 正文中的时间描述（"三天后"、"半年前"等）

检查：
  - 事件顺序是否矛盾？
  - 时间跨度是否合理？（如三天内发生了一个月的事）
  - 角色年龄是否与时间线吻合？
  - 地理距离与旅行时间是否合理？

输出：
  - 发现的时间线矛盾列表（含章节号和具体矛盾）
```

### 审查 4：伏笔回收率

```
输入：
  - 03-追踪/伏笔账本.md

统计：
  - 总伏笔数（含已回收）
  - 已回收数 / 回收率
  - 各紧迫度级别的分布
  - [紧急回收] 列表（需要尽快处理的）
  - 超过 100 章未处理的伏笔（建议放弃或强制回收）

输出：
  | 状态 | 数量 | 占比 |
  |------|------|------|
  | 已回收 | X | X% |
  | 紧急回收 | X | X% |
  | 可回收 | X | X% |
  | 铺垫中 | X | X% |
  | 潜伏 | X | X% |

  需要在接下来 10 章内处理的伏笔：
  1. ...
  2. ...
```

### 审查 5：节奏分布

```
输入：
  - 03-追踪/章节摘要.md（识别爽点/高潮/平淡段）
  - 05-审查/ 下的审查报告（提取爽点和钩子得分）

检查：
  - 爽点间隔：是否超过 5 章无爽点？
  - 高潮分布：每 15-20 章是否有一个中等高潮？
  - 连续平淡：是否有连续 3+ 章节奏评分 < 3/5？
  - 连续高压：是否有连续 5+ 章全是高强度（读者疲劳）

输出：
  节奏图（文字版）：
  第1-10章：●●●○●●○●●● （●=有爽点，○=平淡）
  第11-20章：○●●●○○●●○●
  ...
  问题区间：第 X-Y 章连续平淡，建议调整
```

### 审查 6：情节线完整性

```
输入：
  - 03-追踪/情节线.md

检查：
  - 主线是否在持续推进？（无进展的章节不超过 10 章）
  - 副线是否有断裂？（状态"进行中"但超过 20 章无进展）
  - 已完结的副线是否有遗留问题？
  - 副线数量是否过多？（同时活跃 > 5 条则建议收束部分）

输出：
  | 线索 | 状态 | 最后进展 | 问题 |
  |------|------|---------|------|
  | 主线 | 正常 | 第X章 | 无 |
  | 副线A | ⚠️ 停滞 | 第Y章 | 20章无进展 |
  | 副线B | 正常 | 第Z章 | 无 |
```

## 输出：全局审查报告

保存到 `{book_dir}/05-审查/全局审查-第{N}章.md`

```markdown
# 全局审查报告

审查范围：第{X}章 - 第{Y}章
审查时间：{日期}

## 总览

| 维度 | 评级 | 问题数 |
|------|------|--------|
| 角色一致 | {高/中/低} | {N} |
| 力量自洽 | {高/中/低} | {N} |
| 时间线 | {高/中/低} | {N} |
| 伏笔回收 | {X}% | 紧急{N}条 |
| 节奏 | {高/中/低} | {N}处 |
| 情节线 | {高/中/低} | {N}条停滞 |

## 需要立即处理的问题
1. [严重] ...
2. [严重] ...

## 建议调整
1. [中等] ...
2. [轻微] ...

## 后续大纲修改建议
- 第{N}章章纲建议增加伏笔{X}的回收
- 副线{Y}建议在第{N}章前收束
- 第{X}-{Y}章节奏偏平淡，建议在第{Z}章安排爽点
```

## 审查后操作

用户看完报告后可以：
- "修改第X章的章纲" → 路由到 story-outline 编辑模式
- "把伏笔X标记为放弃" → Tracker 将该伏笔移入归档
- "调整后续节奏" → 修改后续章纲的爽点/钩子安排
- "继续写" → 带着审查结论继续日更
