# Story Deslop - 去AI味

## 前置检查

1. `.story-state` 存在
2. 确定目标：用户指定了章节号，还是处理最新一章？

## 模式

### 模式一：单章处理（默认）

```
Step 1: 确定目标文件
  - 用户说"去AI味" → 处理最新一章（.story-state 的 current_chapter）
  - 用户说"把第15章去一下" → 处理指定章节
  - 定位文件：{book_dir}/04-正文/卷{V}/第{N}章.md

Step 2: 读取参考资料
  - Read references/anti-ai-rules.md（禁用词表+门禁规则）
  - Read {book_dir}/00-设定/风格规范.md（保留有意的风格元素）

Step 3: 调用 Polisher Agent
  - 输入：章节原文 + anti-ai-rules + 风格规范
  - Polisher 执行六大门禁（逐级处理）
  - 输出：润色后正文 + 润色报告

Step 4: 展示对比
  输出给用户：
  - 润色报告（各门禁检测/修改数量）
  - 修改摘要（列出主要改动点）
  - 总修改率

Step 5: 用户确认
  - "接受" → 覆写原文件
  - "部分接受" → 用户指出哪些改动保留、哪些回退
  - "拒绝" → 不修改原文件
```

### 模式二：批量处理

用户说"把第1-10章都去一下AI味"或"整卷去味"

```
Step 1: 确定范围
  - "第1-10章" → 处理第1到第10章
  - "整卷去味" → 处理当前卷所有已写章节
  - "全书去味" → 处理所有已写章节（谨慎，耗时长）

Step 2: 逐章执行
  for each chapter in range:
    调用 Polisher Agent 处理
    输出简要报告（总修改率）
    直接覆写（批量模式不逐章确认）

Step 3: 输出批量报告
  | 章节 | 检测问题数 | 修改数 | 修改率 |
  |------|-----------|--------|--------|
  | 第1章 | 15 | 12 | 8% |
  | 第2章 | 22 | 18 | 11% |
  | ... |

  总计修改 {N} 处，平均修改率 {X}%
```

### 模式三：诊断模式（只检测不修改）

用户说"检测一下AI味"或"这章AI味重吗"

```
Step 1: 读取目标章节
Step 2: 调用 Polisher Agent 仅执行检测（不修改）
Step 3: 输出诊断报告
  - 各门禁的检测命中数
  - AI味严重程度评级（轻度 <10处 / 中度 10-20处 / 重度 >20处）
  - 最典型的3个AI味问题（带原文引用）
  - 建议：是否需要去味处理
```

## 与 story-write 的集成

在 story-write 的流水线中，Stage 3（审查通过）后自动调用 Polisher：
- 不需要用户确认（写作流水线中自动执行）
- 直接修改后进入 Stage 4（保存）
- 在 Stage 6 输出时附带润色报告摘要
