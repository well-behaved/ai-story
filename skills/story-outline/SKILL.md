# Story Outline - 大纲生成

## 前置检查

1. `.story-state` 必须存在（否则引导到 story-start）
2. 读取 `.story-state` 确定当前状态

## 自动模式

如果用户说"自动写大纲"或"快速生成大纲"，或 .story-state 中 `auto_mode == true`：
- 跳过所有"暂停让用户审核"的步骤
- 默认第一卷 50 章（不询问）
- 全部生成完后做一次总确认

## 模式判断

- `status == "initialized"` 且无总纲 → 全新大纲模式
- 用户说"开始新卷" → 新卷生成模式
- 用户说"修改大纲" → 大纲编辑模式

---

## 模式一：全新大纲

### Step 1: 生成总纲

调用 Planner Agent，读取 `00-设定/` 全部 + `01-角色/` 全部。

总纲内容（2000-3000字）：
```markdown
# 总纲

## 核心主线
{一句话概括全书主线}

## 全书走向
### 第一阶段：{阶段名}（约第1-X卷）
{描述}

### 第二阶段：{阶段名}（约第X-Y卷）
{描述}

### 第三阶段：{阶段名}（约第Y-Z卷）
{描述}

## 大转折点（3-5个）
1. 第{N}章左右：{转折描述}
2. ...

## 结局走向
{结局方向描述，不需要太具体}
```

写入 `{书名}/02-大纲/总纲.md`。

**暂停让用户审核**。用户可以修改走向、调整转折点、更改结局方向。

### Step 2: 确定第一卷容量

询问用户：第一卷多少章？
- 建议：50-100 章（番茄标准卷长度）
- 默认 50 章如果用户不确定

### Step 3: 生成卷纲

创建 `{书名}/02-大纲/卷01/` 目录。

卷纲内容（1000-2000字）：
```markdown
# 卷01 {卷名}

## 本卷主线
{本卷要推进的核心事件}

## 本卷副线
- 副线A：{描述}
- 副线B：{描述}

## 节奏规划
- 前10章：设定介绍 + 金手指展示 + 第一次打脸
- 中间章节：升级 + 交友 + 暗线铺垫
- 后10章：本卷高潮 + 大boss战 + 新卷伏笔

## 本卷结尾
{本卷如何收尾，留什么悬念给下一卷}
```

写入 `{书名}/02-大纲/卷01/卷纲.md`。

### Step 4: 批量生成章纲

创建 `{书名}/02-大纲/卷01/章纲/` 目录。

逐章生成章纲（使用 Planner Agent 章纲格式，含字数预算段），每章 300-400 字。

生成策略：
- 前3章（黄金三章）重点打磨：强钩子、快节奏、金手指展示
- 每5章一个小高潮
- 每15-20章一个中等高潮
- 卷末5章是本卷大高潮

写入 `{书名}/02-大纲/卷01/章纲/第001章.md` ~ `第{N}章.md`。

**暂停让用户审核章纲**。

### Step 5: 更新 .story-state

```json
{
  "current_volume": 1,
  "chapters_planned": {N},
  "status": "outlined",
  "volumes": [
    {"vol": 1, "chapters": {N}, "words": 0}
  ]
}
```

注意：`volumes` 数组在此处初始化（第一卷），后续新卷模式 Step 5 追加新条目。story-write 的 Stage 6 卷末检测依赖此数组。

### Step 6: 完成提示

```
✅ 大纲生成完成

  总纲：02-大纲/总纲.md
  卷01纲：02-大纲/卷01/卷纲.md
  章纲：02-大纲/卷01/章纲/第001章.md ~ 第{N}章.md

下一步：说"日更"或"写第一章"开始创作
```

## 模式二：新卷生成

**触发**：用户说"开始新卷"，或当前卷所有章纲写完后自动建议。

### Step 1: 压缩上一卷数据

**1a. 摘要压缩**

读取 `03-追踪/章节摘要.md`，将上一卷的所有章节摘要压缩为卷级摘要：

```
操作：
  1. 读取上一卷所有章节的摘要
  2. 生成一段 800-1000 字的卷级摘要，包含：
     - 本卷核心事件（3-5个）
     - 主角状态变化（起始→结束）
     - 关键角色关系变化
     - 未回收伏笔列表（简明）
  3. 将卷级摘要写入 03-追踪/章节摘要.md 的"卷级摘要"区域
  4. 删除上一卷的详细/中等/一句话摘要（已浓缩到卷级摘要中）
```

**1b. 伏笔归档**

读取 `03-追踪/伏笔账本.md`：
```
操作：
  1. 将"已回收归档"区域的内容移入 03-追踪/归档/伏笔-卷{V}.md
  2. 清空伏笔账本中的"已回收归档"区域
  3. 检查 [潜伏] 区域：超过 100 章未动的伏笔标记为 [紧急回收]
  4. 输出伏笔盘点报告：活跃X条，本卷新增X条，本卷回收X条
```

**1c. 角色状态清理**

读取 `03-追踪/角色状态.md`：
```
操作：
  1. 扫描所有角色，检查最近变化章节号
  2. 连续 2 卷未出场的角色（最近变化章节在上上卷或更早）：
     - 移入 03-追踪/归档/角色-非活跃.md
     - 从角色状态.md 中删除
  3. 不自动归档只缺席 1 卷的角色（可能只是暂时不出场）
  4. 主角团永远不归档
```

**1d. 创建归档目录**（如不存在）

```
03-追踪/归档/
├── 伏笔-卷01.md
├── 伏笔-卷02.md
├── 角色-非活跃.md
└── ...
```

### Step 2: 规划新卷

如果 auto_mode == true，从总纲自动推导新卷方向。否则询问用户：

```
问题：
  1. 新卷的核心事件/目标是什么？（或"按总纲自动推导"）
  2. 有新地图/新势力加入吗？
  3. 主角目标是否转变？
  4. 新卷预计多少章？（默认 50-100 章）
```

### Step 3: 生成新卷卷纲

调用 Planner Agent，输入：
- 总纲中下一阶段的走向
- 上一卷的卷级摘要（刚压缩的 800-1000 字）
- 当前角色状态（活跃角色）
- 活跃伏笔列表（需要在新卷处理的标为重点）
- 用户的方向指引（Step 2 的回答）

产出：`02-大纲/卷{V+1}/卷纲.md`

### Step 4: 生成新卷章纲

流程同全新大纲的 Step 4（批量生成），但额外约束：
- 前 5 章必须安排 [紧急回收] 伏笔的处理
- 第一章要有承接上卷结尾的衔接
- 爽点节奏从本卷第一章就要启动（不要慢热）

产出：`02-大纲/卷{V+1}/章纲/第001章.md` ~ `第{N}章.md`

**章节编号策略**：
- 卷内编号从 001 重新开始（文件名：`第001章.md`）
- 全书连续编号 = 前几卷章数之和 + 卷内章号（从 volumes 数组累加）
  - 例：卷01有100章，卷02第1章 = 全书第101章
  - 例：卷01有80章+卷02有60章，卷03第1章 = 全书第141章
- .story-state 的 `current_chapter` 始终是全书连续编号
- 卷内章号 = current_chapter - 前几卷章数之和

### Step 5: 更新 .story-state

```json
{
  "current_volume": "{V+1}",
  "chapters_planned": "{新卷章数}",
  "status": "outlined",
  "volumes": [
    {"vol": 1, "chapters": 100, "words": 280000, "summary_file": "03-追踪/归档/卷01-摘要.md"},
    {"vol": 2, "chapters": "{新卷章数}", "words": 0}
  ]
}
```

新增 `volumes` 数组记录每卷信息，用于：
- 分卷切换时定位上卷最后一章
- 统计全书数据
- story-write P2 卷首章衔接

### Step 6: 完成提示

```
✅ 第{V+1}卷 规划完成

  上一卷压缩：摘要 {原字数}字 → {800}字 | 归档伏笔 {N}条 | 归档角色 {M}个
  新卷：02-大纲/卷{V+1}/卷纲.md + {N}章章纲
  活跃伏笔：{X}条（其中紧急回收{Y}条，已安排在前5章处理）

下一步：说"日更"继续写作
```

## 模式三：大纲编辑

用户可以说：
- "修改第X章的章纲" → 直接编辑对应文件
- "在第X章后插入一章" → 插入新章纲，后续编号重排
- "删除第X章" → 删除并重排
- "修改卷纲" → 编辑卷纲文件

编辑后提醒用户检查后续章纲是否需要连锁调整。
