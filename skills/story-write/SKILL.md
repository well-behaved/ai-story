# Story Write - 章节写作

这是整个系统的核心 Skill，实现完整的单章写作流水线。

## 前置检查

1. `.story-state` 存在且 `status` 为 "outlined" 或 "writing"
   - 不存在 → 提示："还没有小说项目，请先说'帮我开一本小说'"
   - status 为 "initialized" → 提示："设定已完成但还没有大纲，请先说'写大纲'"
2. 确定当前进度：`book_dir`、`current_volume(V)`、`current_chapter`
3. 下一章编号：`next = current_chapter + 1`
4. 检查章纲文件存在：`02-大纲/卷{V}/章纲/第{next}章.md`
   - 不存在 → 提示："第{next}章的章纲不存在，请先说'写大纲'生成章纲"
5. 检查脏状态：如果 .story-state 中 `writing_in_progress` 已存在，说明上次写作被中断
   - 提示："上次写第{N}章在{stage}阶段被中断，是否从该阶段继续？"
   - 用户确认继续 → 从对应 Stage 恢复
   - 用户说"重新写" → 清除 writing_in_progress，从头开始
6. 标记写作进行中：在 .story-state 中设置 `"writing_in_progress": {"chapter": next, "stage": "assembling"}`

**writing_in_progress.stage 转换时机**：
- `"assembling"` — Stage 1 开始时设置
- `"drafting"` — Stage 2 开始时更新
- `"reviewing"` — Stage 3 开始时更新
- `"polishing"` — Stage 3.5 开始时更新
- `"saved"` — Stage 4 保存正文后更新
- 删除 — Stage 5 Tracker 全部完成后删除（标记本章完成）

## 写作模式

- **单章模式**（默认）："写下一章"、"日更"
- **多章模式**："写3章"、"写到第20章" → 逐章串行执行，每章完成后自动继续
- **指定章节**："写第15章" → 跳转到指定章节（仅限未写过的章节）

## 完整单章流水线

### Stage 1: 上下文组装（Planner Agent）

读取 `.story-state` 获取 `book_dir`、`current_volume(V)`、`current_chapter`。

按 P0-P5 优先级组装：

```
P0（骨架，~3000字）：
  Read {book_dir}/00-设定/书名与简介.md → 只取"一句话简介"（~30字）
  Read {book_dir}/02-大纲/总纲.md → 只提取以下段落（~500字以内）：
    - "## 核心主线"段落（全文）
    - "## 全书走向"中与当前卷对应的阶段描述（只取当前阶段，不取全部）
    - 不取"大转折点"和"结局走向"（避免剧透影响写作自然感）
    - 回退：如果总纲格式非标准（缺少以上标题），加载总纲全文截断到500字
  Read {book_dir}/02-大纲/卷{V}/卷纲.md → 全文（~1500字）
  Read {book_dir}/02-大纲/卷{V}/章纲/第{next}章.md → 全文（~300字）

P1（角色，~1500字）：
  从章纲"出场角色"段落提取角色名列表
  对每个角色：Read {book_dir}/01-角色/{角色名}.md → 取"当前状态"段落
  主角始终加载（即使章纲未列出）

P2（衔接，~2500字）：
  如果 next == 1（全书第一章）：
    P2 为空，跳过（无前章可衔接）
  如果是卷首章（next > 1 且 next == 前几卷章数之和 + 1，从 volumes 数组累加）：
    用 Glob 扫描 04-正文/卷{V-1}/ 找到最后一个章节文件
    Read 该文件 → 取最后 1000 字
  否则（正常情况）：
    Read {book_dir}/04-正文/卷{V}/第{next-1}章.md → 全文

P3（伏笔，~1000字，硬上限 20 条）：
  Read {book_dir}/03-追踪/伏笔账本.md
  按紧迫度排序：[紧急回收] > [可回收] > [铺垫中]
  最多加载 20 条最紧迫的伏笔（每条完整描述）
  超出 20 条的：只加载伏笔标题（每条 ~10 字），让 Writer 知道存在但不看详情
  [潜伏] 的不加载

P4（近期上下文，~1500字）：
  Read {book_dir}/03-追踪/章节摘要.md → 只从"详细区"取最近5章摘要（~1250字）
  不加载中等区/一句话区/卷级摘要（这些在全局审查时才用）

P5（约束，~500字）：
  Read {book_dir}/00-设定/风格规范.md → 全文
  如果 next <= 3（黄金三章）：
    额外加载 references/writing-techniques/黄金三章.md 中对应章节的要求
    （第1章要求 / 第2章要求 / 第3章要求，各约200字）
    注入给 Writer 作为额外硬约束
```

**Token 预算检查**：
- 计算总字数
- 如果 > 12000 字：按 P5→P4→P3 顺序截断
- P0+P1+P2 永远不截断（这是写作的最低需求）

组装完成后，将所有内容打包为一个结构化的上下文块，交给 Writer Agent。

### Stage 2: 初稿生成（Writer Agent）

将组装好的上下文交给 Writer Agent，指令：

```
你现在要写《{书名}》第{V}卷第{next}章。

[P0-P5 上下文内容]

请根据章纲写这一章的正文。目标字数：2000-4000字。
```

Writer Agent 输出初稿正文。

**字数检查**：
- < 1500 字 → 要求 Writer 扩展
- > 5000 字 → 要求 Writer 精简
- 1500-5000 字 → 继续

### Stage 3: 审查评分（Reviewer Agent）

将初稿 + 章纲 + 前章正文 + 角色状态 交给 Reviewer Agent。

Reviewer 输出 10 维评分报告。

**判定逻辑**：
```
第 1 轮：
  >= 35 分 → 通过，进入 Stage 4
  < 35 分 → 将审查报告交给 Writer，要求修改

第 2 轮（如果有）：
  >= 30 分 → 通过（降低标准）
  < 30 分 → 继续修改

第 3 轮（如果有）：
  无论分数 → 通过，但标记"⚠️ 审查未完全通过，建议人工检查"
```

审查循环最多 3 轮，避免无限循环。

**审查报告持久化**：每次审查的最终报告保存到 `{book_dir}/05-审查/第{next}章-审查.md`，用于后续质量趋势分析。

### Stage 3.5: 润色去味（Polisher Agent）

将审查通过的稿件交给 Polisher Agent 执行六大门禁：

```
输入：审查通过的正文 + references/anti-ai-rules.md + 风格规范
处理：门禁1(禁用词) → 门禁2(句式) → 门禁3(心理外化) → 门禁4(节奏) → 门禁5(对话) → 门禁6(结尾)
约束：修改量 <= 15%，不改变情节内容
输出：润色后正文 + 润色报告（各门禁修改数量）
```

更新 .story-state 中 `writing_in_progress.stage` 为 "polishing"。

### Stage 4: 保存终稿（先于追踪更新，确保正文不丢）

确保目录存在：`{book_dir}/04-正文/卷{V}/`

Write 终稿到：`{book_dir}/04-正文/卷{V}/第{next}章.md`

格式：
```markdown
# 第{next}章 {章名}

{正文内容}
```

更新 .story-state 中 `writing_in_progress.stage` 为 "saved"。

### Stage 5: 状态更新（Tracker Agent）

将终稿交给 Tracker Agent，指令：

```
以下是《{书名}》第{next}章的终稿。请提取事实变更并更新追踪文件。

[终稿正文]

当前追踪文件：
[角色状态.md 内容]
[伏笔账本.md 内容]
[章节摘要.md 最近部分]
[情节线.md 内容]
```

Tracker Agent 执行：
1. 提取 6 类事实变更
2. Edit 角色状态.md（快照覆写）
3. Edit 世界状态.md（仅在有变化时）
4. Edit 伏笔账本.md（新增/回收/升级紧迫度）
5. 追加章节摘要（200-300字）
6. Edit 情节线.md（更新进度）
7. Edit .story-state：
   - `current_chapter++`
   - `total_words += 本章字数`
   - `status` 设为 "writing"（如果之前是 "outlined"）
   - 删除 `writing_in_progress` 字段（标记完成）
   - 将本章审查分数追加到 `recent_scores` 数组（保留最近10章）

### Stage 6: 自动触发检查 + 输出给用户

**自动触发检查**（Stage 5 完成后、输出前执行）：

```
检查 1：连续低分预警
  读取 .story-state 的 recent_scores 数组
  如果最近 3 章分数全部 < 40：
    输出：⚠️ 连续 3 章审查低于 40 分，建议执行"全局审查"检查一致性
  如果最近 3 章分数全部 < 35：
    输出：🚨 质量严重下滑，强烈建议暂停写作，先执行"全局审查"

检查 2：定期审查提醒
  如果 current_chapter 是 50 的倍数：
    输出：📋 已写满 {N} 章，建议执行"全局审查"检查一致性和伏笔回收

检查 3：卷末提醒
  如果本章是当前卷最后一章（current_chapter == volumes[V].chapters 对应的全书章号）：
    触发卷末处理（见下方"卷末处理"段落）
```

**输出给用户**：

```
✅ 第{next}章 已完成

  字数：{字数}
  审查评分：{分数}/50
  润色：{修改数}处，修改率{X}%
  {如果有审查警告：⚠️ ...}
  {如果有自动触发提醒：显示在这里}

  已更新：角色状态 | 伏笔账本 | 章节摘要 | 情节线

  下一步：
  - "继续" → 写第{next+1}章
  - "去AI味" → 对本章再次进行去AI味（如仍觉得AI味重）
  - "全局审查" → 检查全书一致性
  - 或直接编辑 04-正文/卷{V}/第{next}章.md 进行手动修改
```

## 多章模式

当用户说"写3章"：

```
for i in 1..3:
  执行完整单章流水线（Stage 1-6）
  每章完成后输出简要进度
  不暂停，直接继续下一章
```

当用户说"写到第20章"：
```
计算需要写多少章（从 current_chapter+1 到 20）
逐章执行
```

**中断**：用户任何时候说话都会中断多章模式，处理用户输入后可以说"继续"恢复。

---

## 卷末处理

当写完当前卷最后一章时：

```
提示用户：
  "第{V}卷 全部{N}章已写完，共{字数}字。
  
  建议：
  1. '全局审查' → 检查本卷一致性
  2. '开始新卷' → 规划下一卷
  3. 继续其他操作"
```

## 快照机制

每 3 章自动保存快照到 `.story-snapshot.json`：

```json
{
  "timestamp": "{时间}",
  "book_name": "{书名}",
  "current_volume": {V},
  "current_chapter": {N},
  "last_action": "completed_chapter_{N}",
  "total_words": {字数}
}
```

用于 compact 后恢复。
