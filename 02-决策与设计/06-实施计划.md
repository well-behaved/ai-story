## 分阶段实施

### Phase 1: MVP（最小可用版本）

**目标**：跑通"立项 → 大纲 → 写第一章 → 自动更新追踪"完整流程

**预计工作量**：1-2 个 session

| 步骤 | 任务 | 产出文件 | 依赖 |
|------|------|---------|------|
| 1.1 | 创建 Skill 目录骨架 | `SKILL.md` + 子目录结构 | 无 |
| 1.2 | 写 SKILL.md 主入口（意图识别+路由） | `SKILL.md` | 1.1 |
| 1.3 | 写 story-setup（初始化部署） | `skills/story-setup/SKILL.md` | 1.1 |
| 1.4 | 写 planner Agent | `agents/planner.md` | 1.1 |
| 1.5 | 写 writer Agent | `agents/writer.md` | 1.1 |
| 1.6 | 写 reviewer Agent | `agents/reviewer.md` | 1.1 |
| 1.7 | 写 tracker Agent | `agents/tracker.md` | 1.1 |
| 1.8 | 写 story-start（5问+设定+角色） | `skills/story-start/SKILL.md` | 1.4 |
| 1.9 | 写 story-outline（总纲+卷纲+章纲） | `skills/story-outline/SKILL.md` | 1.4 |
| 1.10 | 写 story-write（核心写作循环） | `skills/story-write/SKILL.md` | 1.4-1.7 |
| 1.11 | 写 session-start Hook | `hooks/session-start.sh` | 1.3 |
| 1.12 | 端到端测试：从零开始写一本小说的第一章 | 测试结果 | 1.1-1.11 |

**验收标准**：
- 能通过 "帮我开一本小说" 启动 5 问流程
- 能生成设定 + 角色 + 大纲 + 章纲
- 能写出第一章并自动更新追踪文件
- 重开 session 后 hook 能恢复进度

---

### Phase 2: 质量提升

**目标**：去 AI 味 + 完善审查 + 防 compact 失忆

**预计工作量**：1 个 session

| 步骤 | 任务 | 产出文件 |
|------|------|---------|
| 2.1 | 写 polisher Agent | `agents/polisher.md` |
| 2.2 | 写 story-deslop（独立去AI味） | `skills/story-deslop/SKILL.md` |
| 2.3 | 完善 10 维审查评分体系 | 更新 `agents/reviewer.md` |
| 2.4 | 写 pre-compact Hook | `hooks/pre-compact.sh` |
| 2.5 | 写 post-compact Hook | `hooks/post-compact.sh` |
| 2.6 | 创建 anti-ai-rules.md（50+禁用词+六大门禁） | `references/anti-ai-rules.md` |
| 2.7 | 创建番茄平台规范 | `references/platform-rules/番茄小说.md` |
| 2.8 | 创建写作技法库 | `references/writing-techniques/*.md` |
| 2.9 | 将 polisher 集成到 story-write 流水线 | 更新 `skills/story-write/SKILL.md` |
| 2.10 | 测试：写 10 章，验证去AI味和审查效果 | 测试结果 |

**验收标准**：
- 每章经过六大门禁处理后 AI 味明显降低
- 10 维审查能给出有意义的评分和修改建议
- compact 后能从快照恢复进度

---

### Phase 3: 超长篇支持

**目标**：支持 100+ 章不退化

**预计工作量**：1 个 session

| 步骤 | 任务 |
|------|------|
| 3.1 | 实现分卷管理（新卷生成 + 旧卷摘要压缩） |
| 3.2 | 实现伏笔紧迫度自动升级逻辑 |
| 3.3 | 实现分层摘要压缩（近10章详细/10-50中等/50+一句话） |
| 3.4 | 写 story-review（全局审查） |
| 3.5 | 实现自动审查触发（连续3章低于40分 / 每50章定期） |
| 3.6 | 测试：模拟写到第 100 章，验证状态文件大小可控 |

**验收标准**：
- 第 100 章的上下文加载量与第 10 章基本一致
- 伏笔能正确追踪和提醒
- 分卷切换流畅

---

### Phase 4: 番茄特化

**目标**：输出符合番茄平台要求的稿件

| 步骤 | 任务 |
|------|------|
| 4.1 | 创建题材模板库（系统文/升级文/重生文/都市文/赘婿文） |
| 4.2 | 实现敏感词检查 |
| 4.3 | 黄金三章特殊优化（前3章有专门的写作策略） |
| 4.4 | 爽点节奏分析（检查爽点密度是否合理） |
| 4.5 | 番茄格式导出（字数/段落/格式适配） |
