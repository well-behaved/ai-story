## 二、6 个子 Skill 详细设计

### 2.1 story-setup（初始化）

**触发**：安装后首次使用，或用户说"初始化"

**职责**：
1. 检测当前目录是否已有小说项目
2. 部署 hooks 到 `.claude/hooks/`
3. 部署 agents 到 `.claude/agents/`
4. 创建 `.claude/settings.local.json` 注册 hooks
5. 写入 CLAUDE.md（项目级指令）
6. 创建 `.story-deployed` 标记文件

**产出**：
```
.claude/
├── hooks/
│   ├── session-start.sh
│   ├── pre-compact.sh
│   └── post-compact.sh
├── agents/
│   ├── planner.md
│   ├── writer.md
│   ├── reviewer.md
│   ├── polisher.md
│   └── tracker.md
├── settings.local.json
CLAUDE.md
.story-deployed
```

### 2.2 story-start（立项 + 设定）

**触发**：用户说"开一本小说"、"新建项目"

**流程**：
```
Step 1: 5问收集
  Q1: 题材（系统文/升级文/重生文/都市文/...）
  Q2: 主角（性别/年龄/初始处境）
  Q3: 核心冲突（要解决什么）
  Q4: 金手指（特殊能力/系统）
  Q5: 基调（搞笑/热血/阴暗/...）

Step 2: 生成设定（调用 planner Agent）
  → 00-设定/书名与简介.md
  → 00-设定/世界观.md
  → 00-设定/力量体系.md

Step 3: 生成角色（调用 planner Agent）
  → 01-角色/主角.md（4层模板）
  → 01-角色/核心配角.md × N

Step 4: 用户审核修改

Step 5: 生成风格规范
  → 00-设定/风格规范.md
  → 含禁用词表、句式偏好、参考风格

Step 6: 初始化追踪文件
  → 03-追踪/角色状态.md
  → 03-追踪/世界状态.md
  → 03-追踪/伏笔账本.md
  → 03-追踪/章节摘要.md
  → 03-追踪/情节线.md

Step 7: 写入 .story-state（JSON，项目元数据）
  {
    "book_name": "...",
    "genre": "...",
    "platform": "fanqie",
    "current_volume": 1,
    "current_chapter": 0,
    "total_words": 0,
    "created_at": "2026-05-12"
  }
```

### 2.3 story-outline（大纲生成）

**触发**：用户说"写大纲"、"规划"、立项完成后自动进入

**流程**：
```
Step 1: 生成总纲（调用 planner Agent）
  读取：00-设定/ 全部 + 01-角色/ 全部
  产出：02-大纲/总纲.md（全书走向，2000-3000字）
  内容：主线冲突→3-5个大转折点→结局走向

Step 2: 用户审核总纲

Step 3: 生成第一卷卷纲
  读取：总纲 + 设定 + 角色
  产出：02-大纲/卷01/卷纲.md（本卷主线/副线/高潮，1000-2000字）

Step 4: 生成第一卷章纲（批量）
  读取：总纲 + 卷纲 + 设定 + 角色
  产出：02-大纲/卷01/章纲/第001章.md ~ 第N章.md
  每章纲 200-300 字，含：
    - 场景描述
    - 出场角色列表（← 关键，用于 P1 按需加载）
    - 核心冲突/事件
    - 情绪弧线（起→终）
    - 章末钩子类型
    - 爽点类型（如有）

Step 5: 用户审核章纲

Step 6: 更新 .story-state
  current_volume = 1, current_chapter = 0, chapters_planned = N
```

**新卷生成**：
```
触发：用户说"开始新卷"，或当前卷写完自动提示
流程：
  1. 压缩上一卷摘要（详细→一句话，卷级摘要）
  2. 归档上一卷已回收伏笔
  3. 生成新卷卷纲
  4. 生成新卷章纲
  5. 更新 .story-state（current_volume++）
```

### 2.4 story-write（章节写作）★ 核心

**触发**：用户说"日更"、"写下一章"、"继续"

**这是整个系统最核心的 Skill，下面详细展开。**

#### 2.4.1 上下文组装（Planner Agent）

```
Step 1: 读取 .story-state 确定当前进度
  → current_volume, current_chapter

Step 2: 确定下一章编号
  → next = current_chapter + 1

Step 3: 检查章纲是否存在
  → 读取 02-大纲/卷{V}/章纲/第{next}章.md
  → 不存在则提示用户先生成章纲

Step 4: 按 P0-P5 优先级组装上下文

  P0（骨架，必须加载）：
    ← 00-设定/书名与简介.md（摘要，500字以内）
    ← 02-大纲/总纲.md（摘要，取核心走向，500字以内）
    ← 02-大纲/卷{V}/卷纲.md（全文，1000-2000字）
    ← 02-大纲/卷{V}/章纲/第{next}章.md（全文，200-300字）

  P1（角色，按需加载）：
    ← 从章纲中提取"出场角色列表"
    ← 逐个读取 01-角色/{角色名}.md 的当前状态段落
    ← 主角始终加载

  P2（衔接，最近正文）：
    ← 04-正文/卷{V}/第{next-1}章.md（全文，2000-3000字）
    ← 如果是卷首章，读上一卷最后一章

  P3（伏笔）：
    ← 03-追踪/伏笔账本.md（只取紧迫度 >= 铺垫的条目）

  P4（近期上下文）：
    ← 03-追踪/章节摘要.md（最近5章的详细摘要）

  P5（约束）：
    ← 00-设定/风格规范.md（全文）

Step 5: Token 预算检查
  目标：总注入 <= 12000 字（~6K token）
  溢出则按 P5→P4→P3 顺序截断
```

#### 2.4.2 初稿生成（Writer Agent）

```
输入：P0-P5 组装好的上下文

系统提示要点：
  - 你是一个网文写手，正在写第{N}章
  - 严格按照章纲的场景/冲突/钩子写作
  - 目标字数：2000-4000字
  - 遵循风格规范中的所有约束
  - 禁止使用禁用词表中的任何词
  - 每章必须有至少一个爽点
  - 章末必须有钩子（悬念/反转/新信息）
  - 不要在正文中出现任何元叙述

输出：初稿正文（Markdown）
```

#### 2.4.3 审查评分（Reviewer Agent）

```
输入：初稿 + 章纲 + 前章正文 + 角色状态 + 风格规范

10维审查：
  1. 叙事逻辑 (5分)  — 因果关系是否合理
  2. 人物一致 (7.5分) — 性格/能力/关系是否吻合设定
  3. 连贯性   (7.5分) — 与前文是否矛盾
  4. 节奏     (5分)  — 张弛是否得当
  5. 对话     (5分)  — 角色声音是否区分
  6. 描写     (5分)  — 画面感和具体性
  7. 去AI味   (5分)  — AI痕迹程度
  8. 钩子     (5分)  — 章末悬念有效性
  9. 字数     (2.5分) — 2000-4000字范围
  10. 爽点    (2.5分) — 是否有情绪高点

总分 50 分
  >= 35 分：通过，进入润色
  25-34 分：返回 Writer 重写，附上审查意见
  < 25 分：返回 Writer 重写，标记"需要重大修改"

最多 3 轮：
  第1轮不通过 → 重写
  第2轮不通过 → 重写（降低要求到30分）
  第3轮不通过 → 标记为"需人工介入"，保存当前版本
```

#### 2.4.4 润色去味（Polisher Agent）

```
输入：通过审查的稿件

六大门禁逐级过滤：

  门禁1-禁用词过滤：扫描替换50+高频AI词
  门禁2-句式打散：消除连续3+句相同结构
  门禁3-心理外化：把"他感到X"改成动作/环境/生理反应
  门禁4-节奏打碎：中等句长占比>60%时插入超短句和长句
  门禁5-对话去腔调：消除礼貌客套/无意义应答/解释性对话
  门禁6-结尾去升华：删除章末哲理总结/人生感悟

修改量不超过原文 15%
```

#### 2.4.5 状态更新（Tracker Agent）

```
输入：终稿正文 + 当前追踪文件

更新操作：
  1. 角色状态.md → Edit（覆写变化角色的快照段落）
  2. 世界状态.md → Edit（仅在有变化时更新）
  3. 伏笔账本.md → 新伏笔添加 + 已回收标记 + 紧迫度更新
  4. 章节摘要.md → 追加本章摘要(200-300字) + 第11章降级压缩
  5. 情节线.md → 更新主线/副线进度
  6. .story-state → current_chapter++, total_words += 本章字数
```

#### 2.4.6 完整单章流水线

```
用户："日更"
  │
  ▼
Planner: 组装上下文 (P0-P5)
  │
  ▼
Writer: 写初稿 ──────────────────┐
  │                              │
  ▼                              │
Reviewer: 10维审查               │
  ├── >= 35分 → 继续             │
  └── < 35分 → 返回 Writer ──────┘ (最多3轮)
  │
  ▼
Polisher: 六大门禁去AI味
  │
  ▼
Tracker: 更新5个追踪文件 + .story-state
  │
  ▼
保存到 04-正文/卷{V}/第{N}章.md
  │
  ▼
输出给用户审阅
```

### 2.5 story-deslop（独立去AI味）

**触发**：用户说"去AI味"、"这章太AI了"

**流程**：
```
Step 1: 确定目标文件（用户指定 或 最新一章）
Step 2: 执行六大门禁
Step 3: 输出对比（修改前 vs 修改后）
Step 4: 用户确认
```

支持批量："把第1-10章都去一下AI味"

### 2.6 story-review（全局审查）

**触发**：用户说"全局审查"、写完一卷、连续3章低于40分

**检查项**：
- 角色性格偏移（对比初始设定）
- 力量体系自洽
- 时间线一致
- 伏笔回收率
- 节奏分布（爽点密度）
- 情节线完整性

**产出**：`05-审查/全局审查-第{N}章.md`

## 三、5 个 Agent 详细设计

### 3.1 Planner Agent

```
职责：项目初始化、大纲生成、上下文组装(P0-P5)、归档指令、新卷规划
规则：不写正文、不改设定（除非用户要求）、严守Token预算
```

### 3.2 Writer Agent

```
职责：写章节初稿、根据审查意见修改
规则：只写正文、遵循章纲、遵守风格规范和禁用词、2000-4000字、必有爽点和钩子
```

### 3.3 Reviewer Agent

```
职责：10维审查评分、全局一致性审查
规则：只读不写、评分给具体扣分原因、全局审查逐项列问题
```

### 3.4 Polisher Agent

```
职责：六大门禁去AI味、语言润色
规则：不改情节只改表达、修改量<=15%、记录修改点
```

### 3.5 Tracker Agent

```
职责：事实提取、更新5追踪文件、伏笔紧迫度升级、定期压缩
规则：快照模式(覆写不追加)、活跃伏笔only、分层压缩摘要
提取类型：角色变化、世界变化、新伏笔、伏笔回收、时间推进、情节进展
```

## 四、3 个 Hooks 设计

### 4.1 session-start

```
触发：会话启动
作用：读取 .story-state，注入进度提示
输出："当前项目：《书名》第X卷第Y章，共Z字。下一步：写第Y+1章。"
```

### 4.2 pre-compact

```
触发：上下文压缩前
作用：保存快照到 .story-snapshot.json
内容：当前章数、最后操作、上下文摘要
```

### 4.3 post-compact

```
触发：上下文压缩后
作用：注入恢复提示，引导读取 .story-state 恢复上下文
```

## 六、文件格式约定

**章纲格式**：
```markdown
# 第{N}章 {章节名}
## 场景
## 出场角色
## 核心事件
## 情绪弧线
## 章末钩子
## 爽点
```

**角色状态快照格式**：
```markdown
## {角色名}
- 当前境界：
- 位置：
- 关系：
- 情绪：
- 持有物：
- 已知秘密：
- 最近变化：第{N}章 {描述}
```

**伏笔账本格式**：
```markdown
## 活跃伏笔
### [紧急回收]
### [可回收]
### [铺垫中]
### [潜伏]
## 已回收归档
```

## 七、实施优先级

### Phase 1: MVP（最小可用）
- SKILL.md 主入口+路由
- story-setup 初始化
- story-start 5问+设定+角色
- story-outline 总纲+卷纲+章纲
- story-write 核心循环（Planner→Writer→Reviewer→Tracker）
- 4个Agent（planner/writer/reviewer/tracker）
- 5个追踪文件基本维护
- session-start Hook

**MVP 目标**：完成"立项→大纲→写第一章→自动更新追踪"

### Phase 2: 质量提升
- polisher Agent + 六大门禁
- story-deslop 独立去AI味
- 10维审查评分完善
- pre-compact / post-compact Hooks
- references 参考资料库

### Phase 3: 超长篇支持
- 分卷管理（新卷生成+旧卷压缩）
- 伏笔紧迫度自动升级
- 分层摘要压缩
- story-review 全局审查
- 50-100章自动审查触发

### Phase 4: 番茄特化
- 番茄格式输出
- 敏感词检查
- 黄金三章优化
- 爽点节奏分析
