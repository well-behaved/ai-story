## 必须修复的问题（Phase 1 阻塞性）

### Fix 1: P2 首章无前文特判
- 问题：story-write 在 next=1 时尝试读第0章，必然失败
- 修复：story-write Stage 1 P2 增加 `if next == 1: P2 为空`

### Fix 2: P3 伏笔硬上限
- 问题：1000章后活跃伏笔150条，P3 = 4500字溢出
- 修复：P3 最多加载20条最紧迫伏笔，超出只加载标题

### Fix 3: .story-state 增加 writing_in_progress
- 问题：中断导致脏状态无检测
- 修复：写章开始标记in_progress，完成后清除

### Fix 4: 保存顺序调整
- 问题：正文在Tracker之后保存，中断丢正文
- 修复：先保存正文→再更新追踪→最后更新.story-state

### Fix 5: current_volume 初始值统一
- 问题：story-start设0，架构设计设1
- 修复：统一为0表示"未开始"，story-outline生成后更新为1

### Fix 6: 补充引用断链的空壳文件
- polisher.md、story-deslop/SKILL.md、story-review/SKILL.md

### Fix 7: hook扩展名统一+Windows兼容
- 文档统一为.sh，同时创建.ps1版本

### Fix 8: 补充 status 状态机
- initialized → outlined → writing → (completed)
- Tracker 在首章写完后更新 status 为 "writing"

### Fix 9: story-write 明确标注 MVP 无 Polisher
- 在流水线描述中说明 Phase 2 才加入

### Fix 10: SKILL.md 路由表补"继续"关键词
