# 目标方案定义

> 本文档定义产品的完整方案，是后续架构设计和实施的依据。
> 读完本文档应该能完全理解"我们要做什么"。

---

## 基本信息

- **产品名称**：ai-story（暂定）
- **一句话定义**：面向番茄小说的 Claude Code 写作 Skill，支持 1000+ 章超长篇网文的全流程 AI 辅助创作
- **目标用户**：想用 AI 辅助写番茄网文的作者（有 Claude Code 使用经验）
- **技术形态**：Claude Code Skill（可通过 npx skills add 安装）
- **目标平台**：番茄小说（首要），后续可扩展到其他网文平台

---

## 核心功能（10大模块）

### 1. 立项与构思

- 市场热门题材分析（基于番茄榜单数据）
- 题材选择建议（系统文/升级文/重生文/都市文等）
- **5问快速启动**：题材类型、主角设定、核心冲突、金手指/系统、情感基调
- 输出：书名方案 + 简介草稿 + 卖点提炼

### 2. 世界观与角色设计

- **世界观系统化清单**：地理/历史/势力/规则/力量体系/经济/文化
- **角色 4 层模板**：
  - 表层：外貌/口癖/习惯动作
  - 性格层：核心特质/缺陷/价值观
  - 动机层：欲望/恐惧/秘密
  - 弧光层：起始状态→终极状态的变化轨迹
- 角色关系网生成
- 力量体系/等级体系设计

### 3. 大纲体系

- **四级递进**：总纲 → 卷纲 → 章组纲 → 章纲
- **总纲**：全书走向、核心主线、3-5个大转折点
- **卷纲**：每卷 50-100 章，本卷主线/副线/高潮/结局
- **章纲**：每章 200-300 字，场景/冲突/转折/钩子
- **分卷管理**：每卷独立目录，不会出现单文件过大

### 4. 风格定义

- 参考作品分析（用户提供喜欢的作品，AI 提取风格特征）
- 风格规范文档生成（句式偏好/节奏模式/叙事视角/用词倾向）
- AI味检测清单（禁用词/禁用句式/禁用结构）
- 风格一致性评分（每章自动检测是否偏移）

### 5. 章节写作

- **三智能体循环**：
  1. Writer：根据章纲 + 上下文写初稿
  2. Style Reviewer：审查文笔/去AI味/风格一致性
  3. Continuity Reviewer：审查连贯性/角色一致/伏笔
- 不达标自动修订（最多 3 轮）
- 达标后输出终稿

### 6. 自动记忆管理

- **5 核心追踪文件**（AI 自动维护，用户可查看修改）：
  1. `角色状态.md` — 所有活跃角色的当前状态快照
  2. `世界状态.md` — 世界/势力/地点的当前状态
  3. `伏笔账本.md` — 活跃伏笔列表 + 紧迫度分级
  4. `章节摘要.md` — 分层压缩的章节摘要
  5. `情节线.md` — 主线/副线当前进度
- 每章写完自动更新
- 定期压缩旧数据

### 7. 10 维审查体系

| 维度 | 权重 | 说明 |
|------|------|------|
| 叙事逻辑 | 10% | 情节因果关系是否合理 |
| 人物一致 | 15% | 性格/能力/关系是否与设定吻合 |
| 连贯性 | 15% | 与前文是否矛盾 |
| 节奏 | 10% | 张弛是否得当，是否注水 |
| 对话 | 10% | 是否符合角色身份，是否自然 |
| 描写 | 10% | 是否具体，是否有画面感 |
| 去AI味 | 10% | 是否有明显的 AI 痕迹 |
| 钩子 | 10% | 章末是否有有效悬念 |
| 字数 | 5% | 是否在目标范围（2000-4000字） |
| 爽点 | 5% | 是否有情绪高点 |

- 满分 50 分，低于 35 分自动触发重写
- 连续 3 章低于 40 分触发全局审查

### 8. 去 AI 味（六大门禁）

1. **禁用词过滤**：删除高频AI词（"值得注意""不禁""竟然""缓缓"等 50+ 词）
2. **句式打散**：消除"主语+状语+谓语"单调结构，引入倒装/省略/碎片句
3. **心理外化**：把"他感到X"改成动作/环境/生理反应映射
4. **节奏打碎**：交替超短句（3-5字）和长句（30+字），避免中等长度均匀排列
5. **对话去腔调**：消除礼貌客套、删除无意义应答词、增加打断/省略/潜台词
6. **结尾去升华**：删除章末的哲理总结/感悟段落/人生感悟

每层独立检查，逐级过滤，不是一次性处理。

### 9. 番茄平台适配

- **格式规范**：段落长度、对话格式、分隔符使用
- **字数控制**：每章 2000-4000 字（番茄最佳区间）
- **敏感词检查**：平台审核敏感词自动检测和替换
- **章末钩子强化**：确保每章最后 2-3 段有悬念/反转/新信息
- **黄金三章**：前三章特别优化（人设+冲突+金手指展示）

### 10. 日更模式

- 说"日更"或"写下一章"自动触发完整写作流程
- 自动确定当前进度，续写下一章
- 每 3 章自动保存快照（防止 compact 后失忆）
- 支持一次生成多章（"写3章"）

## 每章写作时的上下文组装（P0-P5 优先级）

| 优先级 | 内容 | 约字数 | 说明 |
|--------|------|--------|------|
| P0 | 项目设定 + 总纲 + 本卷卷纲 + 本章章纲 | ~3000 | 骨架，永远加载 |
| P1 | 本章出场角色档案（当前状态） | ~1000-2000 | 按需加载，章纲标注的角色 |
| P2 | 前 1 章完整正文 | ~2000-3000 | 衔接上文，保证语感连续 |
| P3 | 活跃伏笔列表 | ~500-1000 | 紧迫度 >= 铺垫的伏笔 |
| P4 | 最近 5 章摘要 | ~1500 | 近期上下文，避免遗忘近事 |
| P5 | 风格规范 + 禁用词 | ~500 | 约束条件，确保风格一致 |
| **总计** | | **~10000-12000** | **约 5-6K token** |

**关键特性**：不随章节数增长。第 1 章和第 1000 章注入量一样。

**溢出策略**：如果总量超过预算，按 P5→P4→P3 顺序截断（骨架和衔接永远保留）。

## 用户操作流程

### Phase 0: 安装（1分钟）

```
方式A：npx skills add ai-story
方式B：手动 clone 到 ~/.claude/skills/ai-story/
```

安装后，在任意目录说"帮我开一本小说"即可启动。

### Phase 1: 立项（5分钟）

```
用户："帮我开一本番茄小说"
  ↓
AI 问5个问题：
  1. 什么题材？（系统文/升级文/重生文/都市文/...）
  2. 主角什么样？（性别/年龄/初始处境）
  3. 核心冲突是什么？（主角要解决什么问题）
  4. 金手指/系统是什么？（主角的特殊能力）
  5. 什么基调？（轻松搞笑/热血燃向/阴暗复仇/...）
  ↓
AI 生成：书名方案 × 3 + 简介草稿 + 卖点分析
  ↓
用户确认或修改
  ↓
AI 创建项目目录结构
```

### Phase 2: 规划（30分钟 - 1小时）

```
AI 生成世界观 + 力量体系
  ↓
用户审核修改
  ↓
AI 生成主角 + 核心配角档案
  ↓
用户审核修改
  ↓
AI 生成总纲（全书走向）
  ↓
用户审核修改
  ↓
AI 生成第一卷细纲（50-100章的章纲）
  ↓
用户确认，准备开写
```

### Phase 3: 日常写作（每天）

```
用户："日更" 或 "写下一章"
  ↓
Planner：确定当前进度，组装上下文（P0-P5）
  ↓
Writer：根据章纲 + 上下文写初稿
  ↓
Reviewer：10维审查评分
  ├── >= 35分 → 继续
  └── < 35分 → 返回Writer重写（最多3轮）
  ↓
Polisher：润色 + 去AI味（六大门禁）
  ↓
Tracker：更新追踪文件（角色状态/伏笔/摘要）
  ↓
输出终稿，用户可选：
  ├── 直接通过 → 保存到正文目录
  ├── 微调 → 用户修改后保存
  └── 重写 → 给出修改方向，重新生成
```

### Phase 4: 定期维护（每 50-100 章）

```
用户："全局审查" 或 AI 自动触发（连续低分时）
  ↓
AI 扫描：
  - 角色一致性（性格/能力是否偏移）
  - 伏笔回收率（哪些该回收了）
  - 时间线矛盾（日期/年龄错误）
  - 节奏分布（爽点密度是否合理）
  - 力量体系自洽性
  ↓
输出：全局审查报告
  ↓
用户决定是否调整后续大纲
```

### Phase 5: 新卷启动（每 100 章左右）

```
用户："开始新卷" 或到达卷末自动触发
  ↓
AI 生成下一卷卷纲
  ↓
用户审核修改
  ↓
AI 生成下一卷所有章纲
  ↓
压缩上一卷的摘要（详细→一句话）
  ↓
归档上一卷的已回收伏笔
  ↓
继续 Phase 3
```

## 8 个已知坑（设计时必须避免）

| # | 坑 | 后果 | 解法 |
|---|---|------|------|
| 1 | 上下文爆炸 | 超出窗口/混入无关信息导致质量下降 | P0-P5 优先级截断，总量恒定 |
| 2 | Compact 后失忆 | Claude Code compact 后丢失写作进度 | pre/post-compact hooks 保存和恢复关键快照 |
| 3 | 并行写章致一致性崩 | 同时写多章导致状态冲突 | 强制串行逐章，一次只写一章 |
| 4 | Bible 被意外修改 | AI 写作时不小心改了设定文件 | 设定文件写保护（Agent 指令中明确禁止） |
| 5 | 伏笔断线 | 埋了伏笔几百章后忘了回收 | 伏笔账本 + 紧迫度分级 + 定期盘点 |
| 6 | 去AI味停留表面 | 只换了禁用词，句式/结构仍然AI味重 | 六大门禁分层处理，每层独立检查 |
| 7 | YAML 状态增加复杂度 | 格式解析错误/嵌套复杂/不直观 | 用纯 Markdown + 文件存在性标记，人类可读 |
| 8 | 过度工程化 | Agent 太多/框架太重/维护困难 | 控制 4-5 个 Agent，不引入额外框架，纯 Claude Code 原生 |

## 非目标（明确不做的事）

1. **不做独立 Web 应用** — 纯 Skill，不需要服务器
2. **不做多用户协作** — 单用户单项目
3. **不做自动发布** — 只生成内容，发布由用户手动操作
4. **不做图片生成** — 纯文字，封面/插图不在范围内
5. **不做付费功能** — 开源免费
6. **不支持英文写作** — 仅中文网文（初期）
7. **不做实时联网** — 不需要爬取网站、不需要API调用
8. **不引入额外运行时** — 不需要 Docker/数据库/Python 环境（scripts 是可选的）

## 成功指标

1. **可用性**：从安装到写出第一章 < 30 分钟
2. **效率**：日更一章（含审查润色）< 10 分钟
3. **质量**：通过番茄审核率 > 90%
4. **一致性**：1000 章后角色性格偏移 < 10%（人工抽检）
5. **去AI味**：AIGC 检测工具检出率 < 30%
