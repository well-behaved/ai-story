# 系统文写作模板

> **加载时机**：当 story-start 流程中用户选择"系统文"题材，或 AI 判断故事构思属于系统流类型时，加载本模板作为写作指导。

---

## 1. 题材定义与核心卖点

系统文是网文中最经典的"金手指"题材之一，也是番茄小说平台长期霸榜的品类。核心设定是：主角获得一个游戏化的系统界面，通过完成任务、获取积分/经验、兑换能力或物品来持续变强。

**读者为什么爱看：**
- **即时反馈的爽感**：每完成一个任务就有明确奖励，读者跟着主角一起"开箱"
- **数值增长的快感**：属性面板上的数字不断增长，激活人类天生的"收集/成长"欲望
- **确定性的成长路径**：系统给出明确目标，读者不用担心主角"练废了"
- **商城/抽奖的期待感**：和玩游戏一样，"下一个能抽到什么"是持续阅读的动力
- **代入感强**：这一代读者玩游戏长大，系统面板天然亲切

**数据表现**：系统文在番茄完读率高、追读稳定，适合长篇连载。新手作者首选题材之一，因为系统本身就是情节推进引擎，不容易写断。

---

## 2. 典型世界观框架

系统文的世界观分两层：**外部世界** + **系统内部世界**。

### 外部世界（三选一）

| 类型 | 特点 | 适用 |
|------|------|------|
| 都市异能 | 现代都市，暗中有超凡力量 | 代入感强，新手友好 |
| 玄幻/仙侠 | 修炼世界，宗门林立 | 读者基数大，升级体系成熟 |
| 末日/游戏 | 末世降临或全球游戏化 | 系统感最强，和题材最契合 |

### 系统内部框架（必须包含）

- **主界面**：显示主角基础属性（力量/敏捷/体质/精神等，用数字量化）
- **任务面板**：主线任务 + 支线任务 + 每日任务（任务是推动情节的引擎）
- **商城系统**：消耗积分兑换物品/技能，分普通/稀有/传说/神话四个层级
- **背包/仓库**：存放已获得的物品
- **成就系统**（可选）：额外的奖励来源，增加趣味性
- **抽奖/转盘**（可选）：增加随机性和"开箱"快感

### 系统经济设计要点

积分经济必须有通胀控制，这是系统文维持长篇的关键：
- **前期**：任务奖励多、商城物品便宜，快速制造爽感，让读者上钩
- **中期**：奖励增长放缓、物品价格提升，制造"差一点就够"的饥渴感
- **后期**：顶级物品天价、需要特殊货币，制造紧迫感和稀缺性

**示例积分体系**：
- 初级任务奖励：100-500 积分，商城基础物品：200-1000 积分
- 中级任务奖励：1000-5000 积分，商城稀有物品：5000-20000 积分
- 高级任务奖励：10000-50000 积分，商城传说物品：100000+ 积分

## 3. 典型角色配置

### 主角
- **起点**：废柴/普通人/落魄者（起点越低，逆袭越爽）
- **性格**：聪明但不张扬，懂得利用系统优势，有底线但不圣母
- **成长曲线**：从畏手畏脚到游刃有余，逐渐学会最大化利用系统

### 系统（拟人化角色）
系统本身要有个性，它不只是一个冷冰冰的界面：
- **毒舌型**：嘴毒但给力，经常吐槽主角，增加喜剧效果
- **傲娇型**：表面嫌弃主角，实际暗中帮忙
- **坑爹型**：经常发奇葩任务，制造意外剧情
- **冷淡型**：纯机械式响应，但偶尔流露"感情"形成反差

### 第一盟友
- 在主角最困难时出现（或被主角救下）
- 作用：见证主角成长、提供情报、在关键时刻辅助
- 不知道系统存在（系统是主角的秘密）

### 第一反派（打脸对象）
- 身份：小范围内的"强者"（班级霸王/小组长/低级管理者）
- 特点：嚣张、目中无人、曾经欺负过主角
- 作用：第一个被打脸的人，让读者获得爽感，同时展示系统的价值

## 4. 典型情节结构

### 开局（第1-5章）：系统降临

```
第1章：展示主角的惨（被欺负/被开除/被分手/遭遇危机）
第1章末-第2章：系统绑定（脑海中出现系统面板）
第2-3章：探索系统功能（面板、任务、商城），接取第一个任务
第3-4章：完成第一个任务，获得第一个奖励（属性提升/技能/物品）
第4-5章：用奖励打脸第一个看不起自己的人
```

**关键**：系统的核心功能必须在第1章就展示。不要花3章铺垫再出系统，读者等不了。

### 发展（第6-50章）：系统升级循环

标准循环模式（每10-15章一个循环）：
```
系统发布新任务 → 主角面临挑战 → 收集信息/准备 → 完成任务（战斗/考验）
→ 获得奖励 → 用奖励解决更大的问题 → 打脸质疑者 → 新的危机出现 → 循环
```

每个循环要有递进：
- 任务难度递增
- 奖励层级递增
- 对手层级递增
- 主角能力层级递增

### 高潮（每50-80章一个大高潮）

大高潮模式：
```
系统发布特殊/隐藏任务 → 任务难度远超当前等级 → 主角被逼入绝境
→ 利用积累的所有资源/技能 → 系统突破/觉醒新功能 → 逆转翻盘
→ 系统大奖励 → 新的世界/层级展开
```

## 5. 典型爽点类型和节奏

### 核心爽点清单

| 爽点类型 | 频率 | 说明 |
|----------|------|------|
| 任务完成+奖励 | 每3-5章 | 系统文的基础爽点，不能断 |
| 属性面板刷新 | 每5-8章 | 展示数值增长，用前后对比 |
| 商城购买/抽奖 | 每8-12章 | "开箱"快感，要有惊喜 |
| 打脸 | 每5-10章 | 被人看不起→展示实力→众人震惊 |
| 系统升级/解锁新功能 | 每20-30章 | 大爽点，给读者"新玩具" |
| 隐藏任务/成就触发 | 每30-50章 | 意外惊喜，制造"赚到了"的感觉 |

### 节奏建议

- **每章至少一个小爽点**（一次属性提升、一句系统提示、一次小胜利）
- **每3章一个中爽点**（完成任务、打脸、获得新技能）
- **每10-15章一个大爽点**（突破等级、解锁新系统功能、商城大件）
- **不要连续超过2章没有系统相关内容**，读者是来看系统的

## 6. 金手指/系统设计建议

### 常见系统类型

- **任务型系统**：以任务为核心驱动，最经典最好写
- **签到型系统**：每日签到获奖励，适合慢热型故事
- **抽奖型系统**：随机性强，适合制造惊喜
- **模拟器系统**：可以模拟未来/平行世界，信息差类
- **词条/天赋系统**：获取各种被动能力词条，搭配组合
- **副本型系统**：进入副本空间完成挑战，适合战斗向

### 系统设计黄金法则

1. **系统功能要简洁**：核心功能不超过5个，读者记不住太多
2. **每个功能要有成长空间**：基础版→进阶版→终极版
3. **系统要有限制**：不是万能的，有冷却时间/使用条件/副作用
4. **系统要有神秘感**：初期只解锁部分功能，后续逐步揭露
5. **系统来源要埋伏笔**：为什么是主角获得？可以是后期大剧情

## 7. 番茄平台注意事项

### 展示规范
- 系统面板用 **【】** 框住，例如：【力量：15（+3）】
- 积分变化要数字化展示：【积分 +500！当前积分：1350】
- 系统提示用统一格式：【叮！检测到宿主完成任务，奖励已发放】
- 读者喜欢"看到数字变大"，每次属性变化都要明确展示

### 数据建议
- 系统文在番茄男频表现极佳，完读率和追读率都高于平均
- 开局5章定生死，系统必须在前2章内出现
- 每章字数建议2000-2500字，不要超过3000字
- 章末钩子可以用"系统发布新任务"或"商城上新"来吸引下一章

### 合规注意
- 系统不能涉及赌博机制（不要用"赌石""赌注"等表述）
- 奖励不能涉及违禁内容
- 系统面板中不要出现"杀人任务"等敏感表述，可以用"击败""制裁"替代

## 8. 避免的陷阱

### 常见写崩原因

1. **系统万能化**：系统什么都能解决，主角没有挑战 → 要给系统设限制和冷却
2. **积分经济崩溃**：奖励给太多导致后期无法控制 → 提前规划好积分体系的膨胀曲线
3. **任务太机械**：每次都是"杀怪→交任务→领奖励" → 任务类型要多样化（收集、探索、社交、解谜）
4. **忘记外部世界**：只写系统不写人物关系和世界 → 系统是工具，故事核心还是人
5. **系统面板刷屏**：每章大段面板数据 → 面板展示要精简，只展示变化的部分
6. **升级太快或太慢**：前者让读者觉得无聊，后者让读者觉得拖沓 → 按照上述节奏控制
7. **系统人格不一致**：系统时而毒舌时而温柔 → 确定一种性格后保持一致
8. **商城物品无用**：买了的东西后面不用 → 每个重要物品都要在后续剧情中发挥作用（契诃夫之枪原则）
9. **开局太慢**：花了5章还没出系统 → 第1章必须出系统，最迟不超过第2章

### 自检清单

写作时定期检查：
- [ ] 最近5章内是否有系统相关内容？
- [ ] 积分收支是否合理（没有凭空出现大量积分）？
- [ ] 主角是否在利用系统而不是被系统牵着走？
- [ ] 是否有足够的打脸/爽点？
- [ ] 系统功能是否保持一致没有前后矛盾？
